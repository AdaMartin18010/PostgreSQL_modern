# 文档格式统一规范

> **创建时间**: 2025年1月
> **规范版本**: v1.0
> **适用范围**: 所有PostgreSQL Modern项目文档

---

## 📋 规范概述

本文档定义了PostgreSQL Modern项目中所有文档的统一格式标准，确保文档风格一致、结构清晰、易于阅读和维护。

---

## 📝 文档头部格式

### 标准格式

```markdown
# 文档标题

> **更新时间**: YYYY年MM月DD日
> **技术版本**: PostgreSQL 14+（或其他版本）
> **文档编号**: XX-XX-XX（如：08-01-01）
```

### 格式要求

1. **文档标题**:
   - 使用一级标题 `#`
   - 标题简洁明了，反映文档核心内容
   - 不使用标点符号结尾

2. **元数据**:
   - 使用引用块格式 `>`
   - 包含三个必需字段：更新时间、技术版本、文档编号
   - 字段顺序固定：更新时间 → 技术版本 → 文档编号

3. **更新时间**:
   - 格式：`YYYY年MM月DD日`
   - 示例：`2025年1月15日`

4. **技术版本**:
   - 格式：`PostgreSQL 14+` 或具体版本号
   - 可包含扩展版本，如：`PostgreSQL 14+, pgvector 0.7.0+`

5. **文档编号**:
   - 格式：`XX-XX-XX`
   - 示例：`08-01-01`（08表示分类，01-01表示具体文档）

## 📑 目录格式

### 标准格式

```markdown
## 📑 目录

- [文档标题](#文档标题)
  - [📑 目录](#-目录)
  - [1. 章节标题](#1-章节标题)
    - [1.1 子章节](#11-子章节)
```

### 格式要求

1. **目录标题**:
   - 使用二级标题 `## 📑 目录`
   - 使用目录图标 📑

2. **目录结构**:
   - 使用无序列表 `-`
   - 嵌套结构使用2个空格缩进
   - 链接使用Markdown链接格式

3. **目录层级**:
   - 最多显示3级目录
   - 保持目录结构清晰

---

## 📊 章节格式

### 章节标题格式

1. **一级章节**:
   ```markdown
   ## 1. 章节标题
   ```

2. **二级章节**:
   ```markdown
   ### 1.1 子章节标题
   ```

3. **三级章节**:
   ```markdown
   #### 1.1.1 子子章节标题
   ```

### 章节编号规则

- 使用数字编号：1, 2, 3...
- 子章节使用点号分隔：1.1, 1.2, 1.3...
- 保持编号连续，不跳跃

---

## 📋 列表格式

### 无序列表

```markdown
- 项目1
- 项目2
  - 子项目1
  - 子项目2
```

### 有序列表

```markdown
1. 第一项
2. 第二项
   1. 子项1
   2. 子项2
```

### 任务列表

```markdown
- [ ] 未完成任务
- [x] 已完成任务
```

---

## 📊 表格格式

### 标准表格格式

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | 数据2 | 数据3 |
| 数据4 | 数据5 | 数据6 |
```

### 表格要求

1. **对齐方式**:
   - 左对齐：默认
   - 右对齐：`:-----:`
   - 居中对齐：`:-----:`

2. **表格内容**:
   - 保持表格简洁
   - 列数不超过6列
   - 内容清晰易读

## 💻 代码块格式

### SQL代码块

```sql
-- SQL代码示例
CREATE TABLE example (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL
);
```

### Python代码块

```python
# Python代码示例
import psycopg2

def example():
    pass
```

### 代码块要求

1. **语言标识**:
   - SQL代码使用 `sql`
   - Python代码使用 `python`
   - 其他语言使用对应标识

2. **代码注释**:
   - 使用中文注释
   - 注释清晰明了
   - 关键代码必须有注释

3. **代码格式**:
   - 保持代码缩进一致
   - 遵循语言规范
   - 代码可运行

## 🔗 链接格式

### 内部链接

```markdown
[链接文本](./相对路径/文件名.md)
[链接文本](../上级目录/文件名.md)
```

### 外部链接

```markdown
[链接文本](https://example.com)
```

### 链接要求

1. **内部链接**:
   - 使用相对路径
   - 路径正确，链接有效

2. **外部链接**:
   - 使用完整URL
   - 链接有效，可访问

### 引用块

```markdown
> 引用内容
> 可以多行
```

### 引用要求

- 使用 `>` 符号
- 引用内容清晰
- 保持引用格式一致

## 🎨 图标使用

### 常用图标

- 📋 文档、清单
- 📑 目录
- 📊 统计、数据
- 💻 代码
- 🔗 链接
- ✅ 完成
- ⏳ 进行中
- ❌ 未完成
- ⚠️ 警告
- 💡 提示
- 🎯 目标
- 📝 笔记
- 🔍 搜索
- 📚 参考资料
- 🎉 庆祝

### 图标使用规则

1. **一致性**:
   - 相同含义使用相同图标
   - 保持图标使用一致

2. **适度使用**:
   - 不要过度使用图标
   - 图标用于增强可读性

## 📚 参考资料格式

### 标准格式

```markdown
## 7. 参考资料

### 7.1 官方文档

- [PostgreSQL官方文档](https://www.postgresql.org/docs/)
- [pgvector文档](https://github.com/pgvector/pgvector)

### 7.2 学术论文

1. Author, A. (2025). Title. Journal, Volume(Issue), Pages.

### 7.3 相关资源

- [相关链接1](https://example.com)
- [相关链接2](https://example.com)
```

### 参考资料要求

1. **分类清晰**:
   - 官方文档
   - 学术论文
   - 相关资源

2. **格式统一**:
   - 使用有序或无序列表
   - 链接格式正确
   - 信息完整

## 📋 代码示例格式

### 标准格式

```markdown
## 8. 完整代码示例

### 8.1 数据表创建

**创建数据表**：

```sql
CREATE TABLE example (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL
);
```

### 8.2 功能实现

**实现功能**：

```python
def example():
    pass
```
```

### 代码示例要求

1. **章节编号**:
   - 使用 `## 8. 完整代码示例`
   - 子章节使用 `### 8.1`, `### 8.2` 等

2. **代码说明**:
   - 每个代码块前有说明
   - 说明使用加粗格式 `**说明**`

3. **代码完整性**:
   - 代码完整可运行
   - 包含必要的注释
   - 代码格式正确

## 📊 分隔线格式

### 标准分隔线

```markdown
---
```

### 分隔线使用规则

1. **使用场景**:
   - 文档头部和正文之间
   - 主要章节之间
   - 文档结尾

2. **格式要求**:
   - 使用三个连字符 `---`
   - 前后各空一行
