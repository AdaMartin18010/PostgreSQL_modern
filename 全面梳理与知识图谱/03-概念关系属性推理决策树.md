## 一、数据库选型决策树

### 1.1 PostgreSQL适用性决策

```mermaid
flowchart TD
    A[需要选择数据库?] --> B{数据类型?}

    B -->|关系型数据| C{事务需求?}
    B -->|文档数据| D{文档结构复杂度?}
    B -->|时序数据| E{数据规模?}
    B -->|图数据| F{图复杂度?}
    B -->|向量数据| G{AI应用?}

    C -->|强ACID| H{扩展需求?}
    C -->|最终一致| I[考虑MongoDB/DynamoDB]

    H -->|单机足够| J[PostgreSQL ✅]
    H -->|水平扩展| K{分布式事务?}

    K -->|需要| L[Citus/Spanner]
    K -->|不需要| M[读写分离+分片]

    D -->|简单JSON| J
    D -->|复杂嵌套| N[MongoDB]

    E -->|小规模| O[PostgreSQL+TimescaleDB]
    E -->|大规模| P[专用时序数据库]

    F -->|关系+图混合| Q[PostgreSQL+AGE]
    F -->|纯图| R[Neo4j]

    G -->|RAG应用| S[PostgreSQL+pgvector]
    G -->|纯向量| T[专用向量数据库]

    J --> U[优势: ACID完整\n扩展丰富\n生态成熟]
    L --> V[优势: 水平扩展\n分布式SQL]
    O --> W[优势: SQL兼容\n时序优化\n压缩存储]
    Q --> X[优势: 统一存储\n混合查询\n事务支持]
    S --> Y[优势: 统一架构\npgvector成熟\nLangChain集成]
```

**决策规则说明**:

| 决策节点 | 条件 | 推荐方案 | 理由 |
|----------|------|----------|------|
| 关系型+强ACID+单机 | 标准OLTP | PostgreSQL | 完整ACID、丰富特性、成熟生态 |
| 关系型+分布式事务 | 金融/电商 | Citus/Spanner | 强一致性、水平扩展 |
| 关系+向量混合 | RAG应用 | PostgreSQL+pgvector | 统一架构、简化运维 |
| 关系+图混合 | 知识图谱 | PostgreSQL+AGE | ACID图事务、混合查询 |

flowchart TD

### 2.2 复合索引列顺序决策

```mermaid
flowchart TD
    A[设计复合索引] --> B{列的选择性?}

    B -->|差异大| C{查询模式?}
    B -->|相近| D[按查询频率排序]

    C -->|等值查询在前| E[高选择性列优先]
    C -->|范围查询在前| F{范围列数量?}

    F -->|单列| G[范围列放最后]
    F -->|多列| H[最常用的范围列优先]

    E --> I[示例: WHERE a=1 AND b>2\n索引: a,b]
    G --> J[示例: WHERE a=1 AND b>2 AND c=3\n索引: c,a,b 或 a,c,b]

    D --> K[原则: 高频查询列优先]

    I --> L{是否需要覆盖索引?}
    J --> L
    K --> L

    L -->|是| M[包含查询所需所有列]
    L -->|否| N[标准复合索引]

    M --> O[优势: 避免回表\n减少IO]
    N --> P[优势: 索引小\n维护成本低]
```

## 三、事务隔离级别选型决策

```mermaid
flowchart TD

    B -->|最高一致性| C[SERIALIZABLE]
    B -->|高一致性| D{是否可重复读?}
    B -->|中等一致性| E{是否允许幻读?}
    B -->|最低一致性| F[READ UNCOMMITTED]

    D -->|需要| G[REPEATABLE READ]
    D -->|不需要| H{是否允许不可重复读?}

    H -->|允许| I[READ COMMITTED]
    H -->|不允许| G

    E -->|允许| I
    E -->|不允许| J{性能要求?}

    J -->|可接受性能损失| C
    J -->|需要高性能| K{使用显式锁?}

    K -->|是| I
    K -->|否| L[REPEATABLE READ + 应用层处理]

    C --> M[适用: 金融交易\n库存扣减\n账户转账]
    G --> N[适用: 报表查询\n数据分析\n批量处理]
    I --> O[适用: 一般OLTP\nWeb应用\n默认推荐]
    F --> P[警告: 极少使用\n可能读到脏数据]
```
