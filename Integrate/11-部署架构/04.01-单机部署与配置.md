## 一、概述

### 1.1 部署目标

PostgreSQL单机部署与配置指南旨在提供完整的单机PostgreSQL部署、配置和优化方案，适用于开发、测试和小型生产环境。

**核心目标**：

- **快速部署**：提供快速部署方案
- **安全配置**：确保数据库安全
- **性能优化**：优化数据库性能
- **易于维护**：简化运维管理

### 1.2 适用场景

**单机部署适用场景**：

- **开发环境**：本地开发和测试
- **测试环境**：功能测试和性能测试
- **小型生产环境**：低流量、小规模应用
- **学习环境**：学习和实验

**不适用场景**：

- 高可用需求的生产环境
- 大规模并发访问
- 需要水平扩展的场景

### 1.3 版本选择

- **PostgreSQL 18.x**：推荐，最新特性
- **PostgreSQL 17.x**：推荐，稳定版本
- **PostgreSQL 16.x**：兼容，长期支持

## 二、安装与初始化

### 2.1 Linux系统安装

**Ubuntu/Debian系统**：

```bash
# 添加PostgreSQL官方APT仓库
sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -
sudo apt-get update

# 安装PostgreSQL 18
sudo apt-get install postgresql-18 postgresql-client-18

# 安装常用扩展
sudo apt-get install postgresql-contrib-18

# 启动服务
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 检查服务状态
sudo systemctl status postgresql
```

**CentOS/RHEL系统**：

```bash
# 安装PostgreSQL官方YUM仓库
sudo yum install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-7-x86_64/pgdg-redhat-repo-latest.noarch.rpm

# 安装PostgreSQL 18
sudo yum install -y postgresql18-server postgresql18

# 初始化数据库
sudo /usr/pgsql-18/bin/postgresql-18-setup initdb

# 启动服务
sudo systemctl start postgresql-18
sudo systemctl enable postgresql-18

# 检查服务状态
sudo systemctl status postgresql-18
```

**验证安装**：

```bash
# 检查PostgreSQL版本
psql --version

# 连接到数据库
sudo -u postgres psql

# 在psql中检查版本
SELECT version();
```

### 2.2 Windows系统安装

**使用安装程序**：

1. 下载PostgreSQL Windows安装程序
   - 访问：<https://www.postgresql.org/download/windows/>
   - 下载PostgreSQL 18安装程序

2. 运行安装程序
   - 选择安装路径（默认：`C:\Program Files\PostgreSQL\18`）
   - 选择数据目录（默认：`C:\Program Files\PostgreSQL\18\data`）
   - 设置超级用户密码
   - 选择端口（默认：5432）
   - 选择区域设置

3. 安装完成后的配置
   - 安装程序会自动创建Windows服务
   - 可以通过服务管理器启动/停止PostgreSQL

**使用包管理器（Chocolatey）**：

```powershell
# 安装Chocolatey（如果未安装）
Set-ExecutionPolicy Bypass -Scope Process -Force
[System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072
iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

# 安装PostgreSQL
choco install postgresql18

# 启动服务
net start postgresql-x64-18
```

### 2.3 macOS系统安装

**使用Homebrew**：

```bash
# 安装Homebrew（如果未安装）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装PostgreSQL
brew install postgresql@18

# 启动服务
brew services start postgresql@18

# 创建数据库集群
initdb /usr/local/var/postgresql@18
```

**使用Postgres.app**：

1. 下载Postgres.app：<https://postgresapp.com/>
2. 拖拽到Applications文件夹
3. 启动应用，自动创建数据库集群

### 2.4 源码编译安装

**编译安装步骤**：

```bash
# 1. 安装依赖
# Ubuntu/Debian
sudo apt-get install build-essential libreadline-dev zlib1g-dev flex bison libxml2-dev libxslt-dev libssl-dev

# CentOS/RHEL
sudo yum install gcc make readline-devel zlib-devel flex bison libxml2-devel libxslt-devel openssl-devel

# 2. 下载源码
wget https://ftp.postgresql.org/pub/source/v18.1/postgresql-18.1.tar.gz
tar -xzf postgresql-18.1.tar.gz
cd postgresql-18.1

# 3. 配置编译选项
# PostgreSQL 18新增：--with-libnuma（NUMA架构支持）
./configure --prefix=/usr/local/pgsql \
            --with-pgport=5432 \
            --with-openssl \
            --with-libxml \
            --with-libxslt \
            --with-libnuma \
            --enable-nls \
            --enable-thread-safety

# PostgreSQL 18编译选项说明：
# --with-libnuma: 启用NUMA（非一致性内存访问）架构支持
#                 适用于多处理器大型服务器，优化内存访问性能
#                 需要系统安装libnuma-dev（Ubuntu/Debian）或numactl-devel（CentOS/RHEL）

# 4. 编译
make -j$(nproc)

# 5. 安装
sudo make install

# 6. 创建用户和目录
sudo adduser postgres
sudo mkdir -p /usr/local/pgsql/data
sudo chown postgres:postgres /usr/local/pgsql/data

# 7. 初始化数据库
sudo -u postgres /usr/local/pgsql/bin/initdb -D /usr/local/pgsql/data
```

### 2.5 数据库初始化

**初始化数据库集群**：

```bash
# 使用initdb初始化
initdb -D /var/lib/postgresql/data \
       --locale=zh_CN.UTF-8 \
       --encoding=UTF8 \
       --data-checksums

# 参数说明：
# -D: 数据目录
# --locale: 区域设置
# --encoding: 字符编码
# --data-checksums: 启用数据校验和（PostgreSQL 9.3+）
```

**初始化后的目录结构**：

```text
data/
├── PG_VERSION          # PostgreSQL版本号
├── pg_hba.conf         # 客户端认证配置
├── pg_ident.conf       # 用户名映射配置
├── postgresql.conf     # 主配置文件
├── postgresql.auto.conf # 自动生成的配置
├── base/               # 数据库文件
├── global/             # 全局系统表
├── pg_wal/             # WAL日志文件
└── pg_tblspc/          # 表空间链接
```

**启动数据库**：

```bash
# 使用pg_ctl启动
pg_ctl -D /var/lib/postgresql/data -l logfile start

# 使用systemd启动（Linux）
sudo systemctl start postgresql

# 使用服务管理器启动（Windows）
net start postgresql-x64-18
```

## 三、基础配置

### 3.1 postgresql.conf配置

**配置文件位置**：

- Linux: `/etc/postgresql/18/main/postgresql.conf` 或 `$PGDATA/postgresql.conf`
- Windows: `C:\Program Files\PostgreSQL\18\data\postgresql.conf`
- macOS: `/usr/local/var/postgresql@18/postgresql.conf`

**基础配置参数**：

```conf
# ============================================
# 连接设置
# ============================================
listen_addresses = 'localhost'          # 监听地址（生产环境改为'*'或具体IP）
port = 5432                             # 监听端口
max_connections = 100                   # 最大连接数

# ============================================
# 内存设置
# ============================================
shared_buffers = 256MB                  # 共享缓冲区（建议为系统内存的25%）
effective_cache_size = 1GB              # 有效缓存大小（建议为系统内存的50-75%）
work_mem = 4MB                          # 工作内存（每个操作）
maintenance_work_mem = 64MB             # 维护工作内存

# ============================================
# WAL设置
# ============================================
wal_level = replica                     # WAL级别（replica/logical）
max_wal_size = 1GB                      # 最大WAL大小
min_wal_size = 80MB                     # 最小WAL大小
checkpoint_timeout = 5min               # 检查点超时
checkpoint_completion_target = 0.9      # 检查点完成目标

# ============================================
# 查询优化
# ============================================
random_page_cost = 4.0                  # 随机页面代价（SSD建议1.1-1.5）
effective_io_concurrency = 1            # 有效IO并发数（SSD建议200+）
default_statistics_target = 100         # 默认统计目标

# ============================================
# 日志设置
# ============================================
logging_collector = on                  # 启用日志收集
log_directory = 'log'                   # 日志目录
log_filename = 'postgresql-%Y-%m-%d_%H%M%S.log'
log_rotation_age = 1d                   # 日志轮转年龄
log_rotation_size = 100MB               # 日志轮转大小
log_min_duration_statement = 1000       # 记录超过1秒的查询（毫秒）
log_line_prefix = '%t [%p]: [%l-1] user=%u,db=%d,app=%a,client=%h '

# ============================================
# 自动维护
# ============================================
autovacuum = on                         # 启用自动VACUUM
autovacuum_max_workers = 3              # 自动VACUUM最大工作进程数
```

**应用配置**：

```bash
# 重新加载配置（无需重启）
sudo systemctl reload postgresql

# 或在psql中执行
SELECT pg_reload_conf();

# 查看当前配置
SHOW all;
SHOW shared_buffers;
```

```sql
-- 重新加载配置并查看当前配置（带错误处理）
DO $$
BEGIN
    BEGIN
        -- 重新加载配置
        PERFORM pg_reload_conf();
        RAISE NOTICE '配置重新加载成功';

        -- 查看当前配置
        RAISE NOTICE '当前配置值:';
        RAISE NOTICE 'shared_buffers: %', current_setting('shared_buffers');
        RAISE NOTICE 'max_connections: %', current_setting('max_connections');
        RAISE NOTICE 'work_mem: %', current_setting('work_mem');
    EXCEPTION
        WHEN OTHERS THEN
            RAISE WARNING '配置操作失败: %', SQLERRM;
            RAISE;
    END;
END $$;

-- 查看所有配置（带性能测试）
EXPLAIN (ANALYZE, BUFFERS, TIMING)
SELECT name, setting, unit, source, context
FROM pg_settings
WHERE name IN ('shared_buffers', 'max_connections', 'work_mem', 'effective_cache_size')
ORDER BY name;
```

### 3.2 pg_hba.conf配置

**配置文件位置**：

- Linux: `/etc/postgresql/18/main/pg_hba.conf` 或 `$PGDATA/pg_hba.conf`
- Windows: `C:\Program Files\PostgreSQL\18\data\pg_hba.conf`

**配置格式**：

```text
# TYPE  DATABASE        USER            ADDRESS                 METHOD

# 本地连接
local   all             postgres                                peer
local   all             all                                     md5

# IPv4本地连接
host    all             all             127.0.0.1/32            md5

# IPv6本地连接
host    all             all             ::1/128                 md5

# 允许特定IP连接
host    all             all             192.168.1.0/24          md5

# SSL连接
hostssl all             all             0.0.0.0/0               md5
```

**认证方法**：

- **trust**：无条件允许连接（不安全，仅用于本地）
- **md5**：使用MD5加密密码
- **scram-sha-256**：使用SCRAM-SHA-256加密（推荐）
- **peer**：使用操作系统用户名认证（仅本地）
- **cert**：使用SSL证书认证

**安全配置示例**：

```text
# 本地连接使用peer认证
local   all             postgres                                peer
local   all             all                                     scram-sha-256

# 远程连接使用SCRAM-SHA-256
host    all             all             0.0.0.0/0               scram-sha-256

# 特定数据库和用户
host    mydb            myuser           192.168.1.0/24          scram-sha-256
```

### 3.3 内存配置

**内存配置原则**：

```conf
# 共享缓冲区（shared_buffers）
# 建议值：系统内存的25%
# 最小：128MB，最大：系统内存的40%
shared_buffers = 256MB                  # 1GB内存系统
shared_buffers = 2GB                    # 8GB内存系统
shared_buffers = 8GB                    # 32GB内存系统

# 有效缓存大小（effective_cache_size）
# 建议值：系统内存的50-75%
effective_cache_size = 1GB              # 1GB内存系统
effective_cache_size = 6GB              # 8GB内存系统
effective_cache_size = 24GB             # 32GB内存系统

# 工作内存（work_mem）
# 建议值：shared_buffers / (max_connections * 2)
# 注意：每个操作可能使用多个work_mem
work_mem = 4MB                          # 100连接
work_mem = 16MB                         # 200连接

# 维护工作内存（maintenance_work_mem）
# 建议值：系统内存的5-10%
maintenance_work_mem = 64MB             # 1GB内存系统
maintenance_work_mem = 512MB            # 8GB内存系统
maintenance_work_mem = 2GB              # 32GB内存系统
```

**内存配置计算**：

```sql
-- 查看系统内存（带错误处理和性能测试）
DO $$
BEGIN
    BEGIN
        RAISE NOTICE '开始查看系统内存配置';
    EXCEPTION
        WHEN OTHERS THEN
            RAISE WARNING '查询准备失败: %', SQLERRM;
            RAISE;
    END;
END $$;

EXPLAIN (ANALYZE, BUFFERS, TIMING)
SELECT setting::bigint / 1024 / 1024 AS shared_buffers_mb
FROM pg_settings WHERE name = 'shared_buffers';

-- 计算work_mem建议值（带错误处理和性能测试）
EXPLAIN (ANALYZE, BUFFERS, TIMING)
SELECT
    (SELECT setting::bigint FROM pg_settings WHERE name = 'shared_buffers')::bigint /
    (SELECT setting::int FROM pg_settings WHERE name = 'max_connections')::int / 2
    AS recommended_work_mem_bytes;
```

### 3.4 I/O配置

**I/O配置参数**：

```conf
# 随机页面代价（random_page_cost）
# HDD: 4.0（默认）
# SSD: 1.1-1.5
# NVMe: 1.0-1.1
random_page_cost = 4.0                  # HDD
random_page_cost = 1.1                  # SSD

# 有效IO并发数（effective_io_concurrency）
# HDD: 1-2
# SSD: 200-300
# NVMe: 300+
effective_io_concurrency = 1            # HDD
effective_io_concurrency = 200          # SSD

# 顺序页面代价（seq_page_cost）
# 通常保持默认值1.0
seq_page_cost = 1.0
```

**检查存储类型**：

```bash
# Linux: 检查磁盘类型
lsblk -d -o name,rota

# rota=1: 机械硬盘（HDD）
# rota=0: 固态硬盘（SSD/NVMe）

# 检查IO调度器
cat /sys/block/sda/queue/scheduler
```

## 四、PostgreSQL 18优化配置

### 4.1 异步I/O配置

**启用异步I/O**：

```conf
# PostgreSQL 18: 异步I/O配置
# ============================================
# I/O方法选择：
# - 'worker': 使用工作进程进行异步I/O（默认，兼容性好）
# - 'io_uring': 使用Linux io_uring接口（需要Linux 5.1+内核，性能更好）
io_method = 'worker'                    # I/O方法：worker或io_uring

# 异步I/O工作进程配置
max_io_workers = 10                      # 最大I/O工作进程数（用于异步I/O）
maintenance_io_workers = 4              # 维护操作I/O工作进程数（VACUUM、CREATE INDEX等）

# 检查系统支持（Linux）
# 检查io_uring支持：
#   ls /sys/fs/io_uring  # 如果存在则支持
# 检查内核版本：
#   uname -r  # 需要5.1+才能使用io_uring
```

**验证异步I/O配置**：


### 10.3 解决方案

**连接数过多**：

```sql
-- 查看连接详情（带错误处理和性能测试）
DO $$
DECLARE
    connection_count INTEGER;
BEGIN
    BEGIN
        -- 统计连接数
        SELECT COUNT(*) INTO connection_count
        FROM pg_stat_activity;

        RAISE NOTICE '当前总连接数: %', connection_count;

        IF connection_count > 80 THEN
            RAISE WARNING '连接数过多 (%%)，建议检查连接池配置', connection_count;
        END IF;
    EXCEPTION
        WHEN OTHERS THEN
            RAISE WARNING '查询连接信息失败: %', SQLERRM;
            RAISE;
    END;
END $$;

EXPLAIN (ANALYZE, BUFFERS, TIMING)
SELECT
    pid,
    usename,
    application_name,
    client_addr,
    state,
    query_start,
    state_change,
    LEFT(query, 100) AS query_preview
FROM pg_stat_activity
ORDER BY state_change DESC;

-- 终止特定连接（带错误处理）
DO $$
DECLARE
    terminated_count INTEGER := 0;
    backend_pid INTEGER;
BEGIN
    BEGIN
        -- 获取当前后端PID
        SELECT pg_backend_pid() INTO backend_pid;

        -- 终止空闲事务连接
        FOR backend_pid IN
            SELECT pid
            FROM pg_stat_activity
            WHERE pid <> pg_backend_pid()
              AND state = 'idle in transaction'
              AND state_change < NOW() - INTERVAL '1 hour'
        LOOP
            BEGIN
                PERFORM pg_terminate_backend(backend_pid);
                terminated_count := terminated_count + 1;
                RAISE NOTICE '已终止连接 PID: %', backend_pid;
            EXCEPTION
                WHEN OTHERS THEN
                    RAISE WARNING '终止连接 PID % 失败: %', backend_pid, SQLERRM;
            END;
        END LOOP;

        RAISE NOTICE '共终止 % 个空闲事务连接', terminated_count;
    EXCEPTION
        WHEN OTHERS THEN
            RAISE WARNING '终止连接操作失败: %', SQLERRM;
            RAISE;
    END;
END $$;
```
