## 1. 官方与主流来源对标

| 对标维度 | 国际标准/最佳实践 | 本项目现状 | 对齐度 |
|----------|-------------------|------------|--------|
| **版本** | PostgreSQL 18.1 (2025-11-13)，Release 18 特性完整 | 已声明 PG 18.1，网络对齐报告已覆盖 AIO/Skip Scan/uuidv7/虚拟生成列等 | 高 |
| **性能调优** | shared_buffers 25% RAM、work_mem 公式、SSD 下 random_page_cost 1.1–1.5、EXPLAIN ANALYZE、并行参数 | 性能调优深入等覆盖内存/检查点/优化器，与 Percona/官方 Performance Tips 一致 | 高 |
| **PG 18 新特性** | io_method、pg_aios、vacuum_max_eager_freeze_failure_rate、VACUUM/ANALYZE ONLY、OAuth、temporal constraints、RETURNING OLD/NEW | 网络对齐报告已列；具体章节中已逐步显式标注 18.1 新增 GUC/视图/行为变更 | 中高 |
| **pgvector/RAG** | HNSW 索引、混合检索(向量+全文)、chunk 策略、嵌入外置、避免库内调 LLM 生产规模化 | AI/ML 与知识图谱模块已覆盖；已补充“库内 API 调用不推荐生产”等国际共识 | 高 |
| **文档体系** | 官方五部分：Tutorial、SQL、Admin、Client、Server Programming；多语言含 postgres.cn/docs | 项目按主题(01–34)划分，与官方结构不同但覆盖面广；可增加与官方章节的交叉引用 | 中 |

**结论**：核心技术与性能实践已与国际主流对齐；差距主要在显式引用国际来源、PG 18.1 子版本与最新 Release Notes 的逐条核对，以及企业/合规/国际化等非功能维度。

## 3. PG 18.1 核对要点（以 Release 18 为基准）

以下为需在版本特性/性能调优/存储与恢复相关文档中显式标注或补充的 PG 18 变更要点：

| 类别 | 要点 | 说明 |
|------|------|------|
| **I/O** | io_method | 启用 AIO 的 GUC，可选 io_method=worker 等 |
| **I/O** | pg_aios 视图 | 系统视图，展示异步 I/O 使用的文件句柄 |
| **I/O** | io_combine_limit / io_max_combine_limit | 控制 AIO 合并请求 |
| **VACUUM** | vacuum_truncate | GUC 控制 VACUUM 是否截断空文件尾 |
| **VACUUM** | VACUUM/ANALYZE ONLY | 仅处理分区表自身、不递归子分区 |
| **VACUUM** | vacuum_max_eager_freeze_failure_rate | 控制积极冻结 all-visible 页的激进程度 |
| **Autovacuum** | autovacuum_worker_slots | 后台 worker 槽位上限；autovacuum_max_workers 可运行时调至该上限 |
| **Autovacuum** | autovacuum_vacuum_max_threshold | 固定死元组数触发阈值（与百分比并存） |
| **EXPLAIN** | EXPLAIN ANALYZE 默认含 BUFFERS | 18 起自动包含 buffer 统计 |
| **EXPLAIN** | 索引查找次数等新输出 | 各索引扫描节点的查找次数等 |
| **监控** | pg_stat_io 字节级、WAL 行等 | read_bytes、write_bytes、extend_bytes；WAL I/O 行 |
| **监控** | pg_stat_get_backend_io() / pg_stat_get_backend_wal() | 每后端 I/O/WAL 统计 |
| **迁移** | initdb 默认开启 data checksums | 可用 --no-data-checksums 关闭 |
| **迁移** | MD5 认证弃用警告 | CREATE/ALTER ROLE 设置 MD5 密码时告警 |

本文档与《国际对齐与可持续推进计划》配套使用；具体章节修订以本表及官方 [Release 18](https://www.postgresql.org/docs/current/release-18.html) 为准。
