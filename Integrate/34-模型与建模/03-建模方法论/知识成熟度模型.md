## 1. 概述

知识成熟度模型用于评估数据建模知识的完整性和可靠性。
该模型借鉴CMMI（能力成熟度模型集成）的思想，将数据建模知识分为5个成熟度等级，
帮助组织识别知识差距，制定提升计划。

**核心价值**:

- **知识评估**：量化评估组织的数据建模知识水平
- **差距识别**：识别知识薄弱环节
- **提升指导**：提供系统化的知识提升路径
- **持续改进**：建立知识管理循环

## 2. 成熟度等级

### 2.1 五级成熟度模型

| 等级 | 名称 | 特征 | 关键指标 |
|------|------|------|---------|
| Level 1 | 初始级 | 知识零散，依赖个人经验 | 无文档，无标准 |
| Level 2 | 管理级 | 有基本文档，开始标准化 | 有文档，有培训 |
| Level 3 | 定义级 | 知识体系化，有标准流程 | 有知识库，有规范 |
| Level 4 | 量化级 | 知识可量化，有度量体系 | 有指标，有分析 |
| Level 5 | 优化级 | 持续优化，知识创新 | 有创新，有改进 |

### 2.2 详细等级说明

**Level 1 - 初始级**:

- 知识存在于个人经验中
- 没有文档化
- 没有标准流程
- 依赖关键人员

**Level 2 - 管理级**:

- 开始文档化关键知识
- 有基本的培训计划
- 有简单的知识共享机制
- 开始建立标准模板

**Level 3 - 定义级**:

- 知识体系完整
- 有标准化的建模流程
- 有知识库和文档库
- 有角色定义和职责划分

**Level 4 - 量化级**:

- 知识可量化评估
- 有知识质量指标
- 有知识使用统计
- 基于数据做决策

**Level 5 - 优化级**:

- 持续知识创新
- 知识自动更新
- 有知识改进机制
- 知识驱动业务创新

## 3. 评估方法

### 3.1 评估维度

**知识评估维度**:

```sql
-- 知识成熟度评估表
CREATE TABLE knowledge_maturity_assessment (
    assessment_id SERIAL PRIMARY KEY,
    organization_name VARCHAR(200) NOT NULL,
    assessment_date DATE NOT NULL,
    -- 知识维度评分（1-5分）
    documentation_score INT CHECK (documentation_score BETWEEN 1 AND 5),
    standardization_score INT CHECK (standardization_score BETWEEN 1 AND 5),
    training_score INT CHECK (training_score BETWEEN 1 AND 5),
    knowledge_base_score INT CHECK (knowledge_base_score BETWEEN 1 AND 5),
    measurement_score INT CHECK (measurement_score BETWEEN 1 AND 5),
    innovation_score INT CHECK (innovation_score BETWEEN 1 AND 5),
    -- 计算总体成熟度
    overall_maturity_level NUMERIC(3,2) GENERATED ALWAYS AS (
        (
            documentation_score +
            standardization_score +
            training_score +
            knowledge_base_score +
            measurement_score +
            innovation_score
        ) / 6.0
    ) STORED,
    -- 成熟度等级
    maturity_level VARCHAR(50) GENERATED ALWAYS AS (
        CASE
            WHEN overall_maturity_level < 1.5 THEN 'Level 1 - 初始级'
            WHEN overall_maturity_level < 2.5 THEN 'Level 2 - 管理级'
            WHEN overall_maturity_level < 3.5 THEN 'Level 3 - 定义级'
            WHEN overall_maturity_level < 4.5 THEN 'Level 4 - 量化级'
            ELSE 'Level 5 - 优化级'
        END
    ) STORED,
    assessor_name VARCHAR(100),
    notes TEXT
);
```

- 鼓励知识创新

```sql
-- 知识提升效果跟踪视图
CREATE VIEW knowledge_improvement_tracking AS
SELECT
    kia.organization_name,
    kia.assessment_date AS baseline_date,
    kia.overall_maturity_level AS baseline_level,
    kia2.assessment_date AS current_date,
    kia2.overall_maturity_level AS current_level,
    kia2.overall_maturity_level - kia.overall_maturity_level AS improvement,
    EXTRACT(EPOCH FROM (kia2.assessment_date - kia.assessment_date)) / 86400 AS days_elapsed
FROM knowledge_maturity_assessment kia
JOIN knowledge_maturity_assessment kia2
    ON kia.organization_name = kia2.organization_name
    AND kia2.assessment_date > kia.assessment_date
WHERE kia.assessment_date = (
    SELECT MIN(assessment_date)
    FROM knowledge_maturity_assessment
    WHERE organization_name = kia.organization_name
);
```

```sql
    log_id SERIAL PRIMARY KEY,
    item_id INT REFERENCES knowledge_item(item_id),
    user_name VARCHAR(100),
    usage_type VARCHAR(50), -- 'view', 'apply', 'reference'
    usage_date TIMESTAMPTZ DEFAULT NOW(),
    feedback_score INT CHECK (feedback_score BETWEEN 1 AND 10)
);

-- 知识成熟度仪表板视图
CREATE VIEW knowledge_maturity_dashboard AS
SELECT
    COUNT(DISTINCT ki.item_id) AS total_knowledge_items,
    COUNT(DISTINCT kul.user_name) AS active_users,
    AVG(ki.quality_score) AS avg_quality_score,
    SUM(kul.view_count) AS total_views,
    SUM(kul.use_count) AS total_uses,
    COUNT(DISTINCT ki.category) AS category_count
FROM knowledge_item ki
LEFT JOIN knowledge_usage_log kul ON ki.item_id = kul.item_id;
```

```sql
    assessee_name,
    knowledge_area,
    AVG(maturity_level) AS avg_maturity_level,
    CASE
        WHEN AVG(maturity_level) >= 4 THEN 'L4-量化管理'
        WHEN AVG(maturity_level) >= 3 THEN 'L3-已定义'

ORDER BY avg_maturity_level DESC;
```

## 7. 常见问题解答 / FAQ

### Q1: 如何评估知识成熟度等级？

**A**: 评估方法：

```sql
-- 评估维度定义
CREATE TABLE maturity_assessment_criteria (
    level INT PRIMARY KEY,
    level_name VARCHAR(50),
    criteria_description TEXT,
    required_evidence JSONB
);

INSERT INTO maturity_assessment_criteria VALUES
    (1, 'L1-初始', '有基本概念理解', '{"documents": 1, "examples": 0}'::jsonb),
    (2, 'L2-可管理', '能够应用知识', '{"documents": 3, "examples": 2}'::jsonb),
    (3, 'L3-已定义', '有标准化实践', '{"documents": 5, "examples": 5, "standards": 1}'::jsonb),
    (4, 'L4-量化管理', '有量化指标和持续改进', '{"documents": 10, "examples": 10, "metrics": 1}'::jsonb),
    (5, 'L5-优化', '持续创新和优化', '{"documents": 20, "examples": 20, "innovations": 1}'::jsonb);

-- 评估函数
CREATE OR REPLACE FUNCTION assess_maturity_level(
    p_assessee_name VARCHAR,
    p_knowledge_area VARCHAR
)
RETURNS INT AS $$
DECLARE
    v_document_count INT;
    v_example_count INT;
    v_level INT := 1;
BEGIN
    -- 统计文档数量
    SELECT COUNT(*) INTO v_document_count
    FROM knowledge_item
    WHERE owner = p_assessee_name AND category = p_knowledge_area;

    -- 统计示例数量
    SELECT COUNT(*) INTO v_example_count
    FROM knowledge_item
    WHERE owner = p_assessee_name
      AND category = p_knowledge_area
      AND item_type = 'example';

    -- 根据标准评估等级
    IF v_document_count >= 20 AND v_example_count >= 20 THEN
        v_level := 5;
    ELSIF v_document_count >= 10 AND v_example_count >= 10 THEN
        v_level := 4;
    ELSIF v_document_count >= 5 AND v_example_count >= 5 THEN
        v_level := 3;
    ELSIF v_document_count >= 3 AND v_example_count >= 2 THEN
        v_level := 2;
    ELSE
        v_level := 1;
    END IF;

    RETURN v_level;
END;
$$ LANGUAGE plpgsql;
```

### Q2: 如何制定知识提升计划？

**A**: 提升计划制定：

```sql
-- 知识差距分析
CREATE VIEW knowledge_gap_analysis AS
SELECT
    assessee_name,
    knowledge_area,
    current_level,
    target_level,
    (target_level - current_level) AS gap,
    CASE
        WHEN (target_level - current_level) = 0 THEN 'No gap'
        WHEN (target_level - current_level) = 1 THEN 'Small gap'
        WHEN (target_level - current_level) = 2 THEN 'Medium gap'
        ELSE 'Large gap'
    END AS gap_size
FROM (
    SELECT
        ka.assessee_name,
        ka.knowledge_area,
        MAX(ka.maturity_level) AS current_level,
        COALESCE(kip.target_level, MAX(ka.maturity_level)) AS target_level
    FROM knowledge_assessment ka
    LEFT JOIN knowledge_improvement_plan kip ON ka.assessee_name = kip.assessee_name
        AND ka.knowledge_area = kip.knowledge_area
    GROUP BY ka.assessee_name, ka.knowledge_area, kip.target_level
) levels;

-- 生成提升计划
CREATE OR REPLACE FUNCTION generate_improvement_plan(
    p_assessee_name VARCHAR,
    p_knowledge_area VARCHAR,
    p_target_level INT
)
RETURNS JSONB AS $$
DECLARE
    v_current_level INT;
    v_gap INT;
    v_plan JSONB;
BEGIN
    -- 获取当前等级
    SELECT MAX(maturity_level) INTO v_current_level
    FROM knowledge_assessment
    WHERE assessee_name = p_assessee_name
      AND knowledge_area = p_knowledge_area;

    v_gap := p_target_level - COALESCE(v_current_level, 0);

    -- 根据差距生成计划
    v_plan := jsonb_build_object(
        'current_level', v_current_level,
        'target_level', p_target_level,
        'gap', v_gap,
        'estimated_months', v_gap * 2,
        'actions', jsonb_build_array(
            jsonb_build_object('action', '学习基础知识', 'duration_weeks', 2),
            jsonb_build_object('action', '实践应用', 'duration_weeks', 4),
            jsonb_build_object('action', '总结优化', 'duration_weeks', 2)
        )
    );

    RETURN v_plan;
END;
$$ LANGUAGE plpgsql;
```

### Q3: 如何跟踪知识成熟度提升？

**A**: 跟踪方法：

```sql
-- 知识成熟度趋势分析
CREATE VIEW maturity_trend_analysis AS
SELECT
    assessee_name,
    knowledge_area,
    assessment_date,
    maturity_level,
    LAG(maturity_level) OVER (PARTITION BY assessee_name, knowledge_area ORDER BY assessment_date) AS previous_level,
    maturity_level - LAG(maturity_level) OVER (PARTITION BY assessee_name, knowledge_area ORDER BY assessment_date) AS level_change,
    assessment_date - LAG(assessment_date) OVER (PARTITION BY assessee_name, knowledge_area ORDER BY assessment_date) AS days_between_assessments
FROM knowledge_assessment
ORDER BY assessee_name, knowledge_area, assessment_date;

-- 提升速度分析
SELECT
    assessee_name,
    knowledge_area,
    COUNT(*) AS assessment_count,
    MAX(maturity_level) - MIN(maturity_level) AS total_improvement,
    MAX(assessment_date) - MIN(assessment_date) AS total_days,
    ROUND((MAX(maturity_level) - MIN(maturity_level))::NUMERIC / NULLIF(EXTRACT(EPOCH FROM (MAX(assessment_date) - MIN(assessment_date))) / 86400, 0) * 30, 2) AS improvement_per_month
FROM knowledge_assessment
GROUP BY assessee_name, knowledge_area
HAVING COUNT(*) > 1
ORDER BY improvement_per_month DESC;
```

### Q4: 知识成熟度如何与项目质量关联？

**A**: 关联分析：

```sql
-- 项目质量与知识成熟度关联
CREATE TABLE project_quality_metrics (
    project_id BIGSERIAL PRIMARY KEY,
    project_name VARCHAR(100) NOT NULL,
    team_maturity_level NUMERIC(3,1),
    code_quality_score INT,
    performance_score INT,
    maintainability_score INT,
    completion_date DATE
);

-- 关联分析查询
SELECT
    CASE
        WHEN team_maturity_level >= 4 THEN 'High Maturity'
        WHEN team_maturity_level >= 3 THEN 'Medium Maturity'
        ELSE 'Low Maturity'
    END AS maturity_category,
    AVG(code_quality_score) AS avg_code_quality,
    AVG(performance_score) AS avg_performance,
    AVG(maintainability_score) AS avg_maintainability,
    COUNT(*) AS project_count
FROM project_quality_metrics
GROUP BY maturity_category
ORDER BY maturity_category;
```

### Q5: 如何建立知识成熟度管理体系？

**A**: 管理体系建立：

```sql
-- 知识成熟度管理体系
CREATE TABLE knowledge_maturity_system (
    system_id BIGSERIAL PRIMARY KEY,
    organization_name VARCHAR(100) NOT NULL,
    knowledge_area VARCHAR(100) NOT NULL,
    current_maturity_level INT NOT NULL,
    target_maturity_level INT NOT NULL,
    governance_structure JSONB,
    measurement_framework JSONB,
    improvement_process JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 管理体系仪表板
CREATE VIEW maturity_management_dashboard AS
SELECT
    organization_name,
    COUNT(DISTINCT knowledge_area) AS knowledge_areas_count,
    AVG(current_maturity_level) AS avg_current_level,
    AVG(target_maturity_level) AS avg_target_level,
    COUNT(*) FILTER (WHERE current_maturity_level >= target_maturity_level) AS achieved_count,
    COUNT(*) FILTER (WHERE current_maturity_level < target_maturity_level) AS gap_count
FROM knowledge_maturity_system
GROUP BY organization_name;
```
