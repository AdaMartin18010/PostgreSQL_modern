# 文档格式规范

> **创建日期**: 2025年1月
> **用途**: 统一所有文档的格式标准
> **状态**: 强制执行

---

## 📋 文档结构标准

### 1. 文档头部

每个文档必须包含以下头部信息：

```markdown
# 文档标题

> **创建日期**: YYYY年MM月
> **来源**: 来源说明
> **状态**: 文档状态
> **文档编号**: XX-XX（可选）
```

---

### 2. 目录（TOC）

每个文档必须包含目录，格式如下：

```markdown
## 📑 目录

- [1. 一级标题](#1-一级标题)
- [2. 二级标题](#2-二级标题)
  - [2.1 子标题](#21-子标题)
  - [2.2 子标题](#22-子标题)
- [3. 三级标题](#3-三级标题)
```

**要求**:

- 目录必须完整覆盖文档所有主要章节
- 使用锚点链接（自动生成）
- 层级不超过3级

---

### 3. 标题编号规范

#### 一级标题（章节）

```markdown
## 1. 章节名称
```

**编号规则**:

- 使用阿拉伯数字：1, 2, 3, ...
- 编号后加句号和空格
- 不使用emoji（目录中可以使用）

---

#### 二级标题（小节）

```markdown
### 1.1 小节名称
```

**编号规则**:

- 格式：`父级编号.子级编号`
- 如：1.1, 1.2, 2.1, 2.2

---

#### 三级标题（子小节）

```markdown
#### 1.1.1 子小节名称
```

**编号规则**:

- 格式：`父级编号.子级编号.孙级编号`
- 如：1.1.1, 1.1.2, 2.1.1

---

### 4. 内容格式规范

#### 代码块

```markdown
```sql
-- SQL代码示例
CREATE TABLE example (...);
```

```

**要求**:
- 指定语言类型（sql, python, json等）
- 添加必要的注释
- 代码可运行

---

#### 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 值1 | 值2 | 值3 |
```

**要求**:

- 表头使用粗体或清晰描述
- 对齐整齐
- 内容简洁

---

#### 列表

**有序列表**:

```markdown
1. 第一项
2. 第二项
3. 第三项
```

**无序列表**:

```markdown
- 项目1
- 项目2
- 项目3
```

---

### 5. 文档尾部

每个文档必须包含：

```markdown
## X. 相关资源

- [相关文档1](./path/to/doc1.md)
- [相关文档2](./path/to/doc2.md)

## X+1. 参考文档

- 《参考书籍》- 作者
- [在线资源](https://example.com)

---

**最后更新**: YYYY年MM月
**维护者**: PostgreSQL Modern Team
```

## 📐 文档编号规则

### 文件夹编号

- `00-导航与索引/`: 00
- `01-数据建模理论基础/`: 01
- `02-权威资源与标准/`: 02
- `03-建模方法论/`: 03
- `04-OLTP建模/`: 04
- `05-OLAP建模/`: 05
- `06-IoT与时序建模/`: 06
- `07-工作流建模/`: 07
- `08-PostgreSQL建模实践/`: 08
- `09-建模模式与反模式/`: 09
- `10-综合应用案例/`: 10

### 文档编号格式

`文件夹编号-文档序号`

**示例**:

- `01-01`: 01文件夹的第1个文档
- `02-03`: 02文件夹的第3个文档
- `08-02`: 08文件夹的第2个文档

## ✅ 检查清单

创建或更新文档时，请检查：

- [ ] 文档头部信息完整
- [ ] 包含完整目录
- [ ] 所有标题都有编号
- [ ] 编号格式正确（1. 1.1 1.1.1）
- [ ] 代码示例可运行
- [ ] 包含相关资源链接
- [ ] 包含参考文档
- [ ] 最后更新日期正确
