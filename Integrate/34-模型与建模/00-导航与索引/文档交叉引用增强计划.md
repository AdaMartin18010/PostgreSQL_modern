## 2. 交叉引用标准

### 2.1 引用分类

**核心相关文档**（必须引用）:

- 同一目录下的相关文档
- 理论基础文档（如范式理论、ER模型）
- 实践文档（如性能优化、索引策略）

**扩展相关文档**（建议引用）:

- 应用案例文档
- 模式与反模式文档
- 方法论文档

**参考资源**（可选引用）:

- 权威资源索引
- 术语对照表
- 快速查找指南

### 2.2 引用格式

```markdown
## X. 相关资源 / Related Resources

### X.1 核心相关文档 / Core Related Documents

- [文档名称](../../path/to/doc.md) - 简短说明
- [文档名称](../other/path/to/doc.md) - 简短说明

### X.2 理论基础 / Theoretical Foundation

- [理论基础文档](../../path/to/theory.md) - 相关理论

### X.3 实践指南 / Practical Guides

- [实践文档](../../path/to/practice.md) - 实践建议

### X.4 应用案例 / Application Cases

- [案例文档](../../path/to/case.md) - 相关案例

### X.5 参考资源 / Reference Resources

- [权威资源索引](../00-导航与索引/权威资源索引.md) - 权威资源列表
- [术语对照表](../00-导航与索引/术语对照表.md) - 术语对照
- PostgreSQL官方文档: [相关主题](https://www.postgresql.org/docs/...)
```

## 4. 检查清单

### 4.1 内容检查

- [ ] 每个文档都有"相关资源"章节
- [ ] 相关资源章节包含至少3-5个链接
- [ ] 链接指向的文档存在且有效
- [ ] 链接说明清晰准确

### 4.2 格式检查

- [ ] 使用统一的引用格式
- [ ] 链接路径正确（相对路径）
- [ ] 说明文字简洁明了
- [ ] 分类清晰（核心、理论、实践、案例）

### 4.3 质量检查

- [ ] 引用的文档确实相关
- [ ] 没有遗漏重要的相关文档
- [ ] 引用关系双向（A引用B，B也引用A）
- [ ] 引用帮助读者理解内容
