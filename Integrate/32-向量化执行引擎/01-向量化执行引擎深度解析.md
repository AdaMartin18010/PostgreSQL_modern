## 一、概述

### 1.1 什么是向量化执行？

向量化执行（Vectorized Execution）是现代OLAP数据库的核心技术，通过**一次处理一批数据（向量）**而非逐行处理，大幅提升CPU效率。

```
传统执行模型 (Tuple-at-a-time):
┌─────────┐   ┌─────────┐   ┌─────────┐
│  Scan   │──►│ Filter  │──►│  Agg    │
└─────────┘   └─────────┘   └─────────┘
     ↑             ↑             ↑
   1行/次        1行/次        1行/次

向量化执行模型 (Vector-at-a-time):
┌─────────┐   ┌─────────┐   ┌─────────┐
│  Scan   │──►│ Filter  │──►│  Agg    │
└─────────┘   └─────────┘   └─────────┘
     ↑             ↑             ↑
  1024行/批    1024行/批    1024行/批
```

### 1.2 为什么向量化更快？

| 优势 | 说明 | 性能收益 |
|------|------|----------|
| **SIMD优化** | 单条指令处理多组数据 | 4-16x |
| **Cache友好** | 减少缓存未命中 | 2-5x |
| **分支预测** | 减少分支预测失败 | 1.5-2x |
| **函数调用** | 减少虚函数调用开销 | 2-3x |

## 二、核心原理

### 2.1 列式存储与PAX格式

```
PAX (Partition Attributes Across):
┌─────────────────────────────────────────────────┐
│ Row Group 0 (1000行)                            │
├─────────────────────────────────────────────────┤
│ ┌──────────┐ ┌──────────┐ ┌──────────┐         │
│ │ Column 0 │ │ Column 1 │ │ Column 2 │         │
│ │ (压缩)   │ │ (压缩)   │ │ (压缩)   │         │
│ └──────────┘ └──────────┘ └──────────┘         │
├─────────────────────────────────────────────────┤
│ Row Group 1 (1000行)                            │
└─────────────────────────────────────────────────┘

优势:
- 列内数据类型相同 → 更好的压缩
- 只需要读取查询涉及的列
- SIMD友好（连续内存访问）
```

### 2.2 向量化算子

```python
# 伪代码：向量化Filter算子
class VectorizedFilter:
    def __init__(self, child, predicate):
        self.child = child
        self.predicate = predicate
        self.batch_size = 1024

    def next(self):
        # 从子算子获取一批数据
        batch = self.child.next()
        if batch is None:
            return None

        # 应用谓词（向量化）
        selection_vector = []
        for i in range(batch.size):
            if self.predicate.evaluate(batch, i):
                selection_vector.append(i)

        # 返回过滤后的批次
        return batch.select(selection_vector)

# SIMD优化版本（使用AVX-512）
class SIMDKernel:
    def compare_int64(self, data, value, result):
        """AVX-512比较操作"""
        # 一次处理8个int64值
        for i in range(0, len(data), 8):
            vec = _mm512_loadu_si512(data[i])
            cmp = _mm512_cmp_epi64_mask(vec, value, _MM_CMPINT_EQ)
            result[i//8] = cmp
```

### 2.3 表达式求值

```
传统表达式树求值:
SELECT a + b * c FROM t

逐行求值:
for each row:
    temp = evaluate(b * c)
    result = evaluate(a + temp)

向量化表达式求值:
1. 加载列b的向量 (1024个值)
2. 加载列c的向量 (1024个值)
3. SIMD乘法: temp_vec = b_vec * c_vec
4. 加载列a的向量 (1024个值)
5. SIMD加法: result_vec = a_vec + temp_vec
```

## 五、向量化实现实战

### 5.1 简单向量化算子实现

```cpp
// C++向量化Filter算子示例
#include <immintrin.h>

class VectorizedFilterInt {
private:
    int* data;
    size_t size;
    int threshold;

public:
    VectorizedFilterInt(int* d, size_t s, int t)
        : data(d), size(s), threshold(t) {}

    // AVX2版本：一次处理8个int32
    std::vector<int> filter_avx2() {
        std::vector<int> result;
        result.reserve(size);

        __m256i threshold_vec = _mm256_set1_epi32(threshold);

        size_t i = 0;
        for (; i + 7 < size; i += 8) {
            __m256i data_vec = _mm256_loadu_si256((__m256i*)&data[i]);
            __m256i cmp = _mm256_cmpgt_epi32(data_vec, threshold_vec);

            int mask = _mm256_movemask_ps((__m256)cmp);

            // 提取匹配的项
            for (int j = 0; j < 8; j++) {
                if (mask & (1 << j)) {
                    result.push_back(data[i + j]);
                }
            }
        }

        // 处理剩余元素
        for (; i < size; i++) {
            if (data[i] > threshold) {
                result.push_back(data[i]);
            }
        }

        return result;
    }
};
```

### 5.2 向量化聚合

```cpp
// 向量化SUM实现
int64_t vectorized_sum(int64_t* data, size_t size) {
    __m512i sum_vec = _mm512_setzero_si512();

    size_t i = 0;
    // AVX-512：一次处理8个int64
    for (; i + 7 < size; i += 8) {
        __m512i data_vec = _mm512_loadu_si512((__m512i*)&data[i]);
        sum_vec = _mm512_add_epi64(sum_vec, data_vec);
    }

    // 水平求和
    int64_t sum = _mm512_reduce_add_epi64(sum_vec);

    // 处理剩余元素
    for (; i < size; i++) {
        sum += data[i];
    }

    return sum;
}
```

## 六、性能对比

### 6.1 基准测试

```
测试场景: 1亿行数据，简单聚合查询
SELECT SUM(amount), AVG(amount), COUNT(*) FROM transactions WHERE category = 'A'

结果:
┌────────────────────┬───────────┬──────────┐
│ 执行方式            │ 时间      │ 相对性能 │
├────────────────────┼───────────┼──────────┤
│ PostgreSQL (标准)   │ 12.5s     │ 1x       │
│ PostgreSQL + JIT    │ 8.3s      │ 1.5x     │
│ DuckDB (向量化)     │ 0.4s      │ 31x      │
│ 自定义C++ SIMD      │ 0.25s     │ 50x      │
└────────────────────┴───────────┴──────────┘
```

### 6.2 适用场景

**向量化执行适合**:

- 大批量数据分析（OLAP）
- 复杂聚合查询
- 列存储数据
- 长时间运行的查询

**不适合**:

- 单行点查（OLTP）
- 小数据量查询
- 需要事务隔离的场景
