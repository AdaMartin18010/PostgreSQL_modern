- 所有架构设计文档

**应用场景**:

- 技术方案对比

**目标文档**:

- 对比分析主题
- 部署方案文档

#### 2.3 决策图网 (Decision Tree)

**应用场景**:

- 技术选型决策

**目标文档**:

- 技术选型指南
- 场景分析文档

- 部署方案文档

#### 2.4 证明图网 (Proof Network)

**应用场景**:

- 形式化论证

**目标文档**:

- 理论体系文档

#### 2.5 架构UML图

**应用场景**:

- 系统架构设计
- 组件关系

- 部署架构

**目标文档**:

- 架构设计文档

## 📊 思维表达方式模板

### 思维导图模板

```mermaid
mindmap
  root((主题名称))
    核心概念1
      子概念1.1
      子概念1.2
    核心概念2
      子概念2.1
      子概念2.2
```

### 多维矩阵模板

| 维度1 | 维度2 | 维度3 | 维度4 | 维度5 |
|------|------|------|------|------|
| 选项A | 值A1 | 值A2 | 值A3 | 值A4 |
| 选项B | 值B1 | 值B2 | 值B3 | 值B4 |

### 决策树模板

```mermaid
flowchart TD
    A[起始决策点] -->|条件1| B[选项A]
    A -->|条件2| C[选项B]
    B -->|条件3| D[结果1]
    B -->|条件4| E[结果2]
```

### 证明图模板

```mermaid
graph TD
    A[前提1] --> C[结论1]
    B[前提2] --> C
    C --> D[结论2]
    E[前提3] --> D
```

## ✅ 质量标准

### 内容质量标准

1. **完整性**: 每个主题必须有完整的知识体系
2. **准确性**: 所有技术信息必须准确
3. **时效性**: 所有版本信息必须更新到最新
4. **可读性**: 文档结构清晰,易于理解

### 思维表达质量标准

1. **准确性**: 图表必须准确反映内容
2. **清晰性**: 图表必须清晰易懂
3. **完整性**: 图表必须覆盖关键概念
4. **美观性**: 图表格式统一,美观大方
