## 1. 概述

PostgreSQL采用开放、透明的社区治理模式。

**治理原则**:

- 开放透明
- 基于共识
- 技术导向
- 社区驱动

---

## 2. 治理模型

### 2.1 治理层次

```
核心团队
  ↓
开发团队
  ↓
社区成员
```

### 2.2 决策机制

```text
- 技术决策：核心团队讨论
- 重大决策：社区投票
- 日常决策：团队协商
```

---

## 3. 决策流程

### 3.1 提案流程

```
1. 提出提案
   ↓
2. 社区讨论
   ↓
3. 技术审查
   ↓
4. 投票决定
   ↓
5. 实施执行
```

### 3.2 投票机制

```text
- 技术提案：核心团队投票
- 社区提案：社区投票
- 发布决策：发布团队决定
```

---

## 4. 角色与职责

### 4.1 核心角色

| 角色 | 职责 |
|------|------|
| **核心团队成员** | 技术决策、代码审查 |
| **提交者** | 代码提交、版本管理 |
| **文档维护者** | 文档更新、翻译 |
| **社区支持者** | 社区支持、问题解答 |

---

## 5. 参与机制

### 5.1 参与方式

```text
1. 邮件列表讨论
2. IRC/Slack交流
3. GitHub贡献
4. 会议参与
```

### 5.2 贡献路径

```
社区成员
  ↓
贡献者
  ↓
维护者
  ↓
核心成员
```
