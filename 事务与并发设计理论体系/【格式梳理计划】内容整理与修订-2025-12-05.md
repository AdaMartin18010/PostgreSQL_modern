## 📐 格式规范标准

### Markdown格式规范

#### 标题层级

```markdown
# 一级标题 (文档标题)

## 二级标题 (主要章节)

### 三级标题 (子章节)

#### 四级标题 (细节)
```

**规则**:

- 一级标题只有一个（文档标题）
- 二级标题是主要章节
- 不超过四级标题

#### 代码块

```markdown
    ```python
    # 必须指定语言
    def example():
        pass
    ```

```

**规则**:

- 所有代码块必须指定语言
- SQL代码用 `sql`
- Rust代码用 `rust`
- Python代码用 `python`
- 文本/伪代码用 `text`

#### 表格格式

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容 | 内容 | 内容 |
```

**规则**:

- 表头与内容对齐
- 使用统一的列分隔符
- 表格前后空行

#### 列表格式

```markdown
- 无序列表项1
- 无序列表项2

1. 有序列表项1
2. 有序列表项2
```

**规则**:

- 列表前后空行
- 嵌套列表统一缩进（2空格或4空格）

#### 链接格式

```markdown
[链接文本](./相对路径/文件名.md)
[外部链接](https://example.com)
```

**规则**:

- 使用相对路径
- 路径大小写敏感
- 检查链接有效性

---

### 文档结构规范

#### 文档头部模板

```markdown
# [编号] | 文档标题

> **模块定位**: 一句话说明定位
> **创建日期**: YYYY-MM-DD
> **文档版本**: x.y.z

---

## 📑 目录

[自动生成或手动维护]

---
```

#### 章节编号规范

```markdown
## 一、大章节

### 1.1 子章节

#### 1.1.1 细节

或者

## 一、大章节

### 1.1 子章节

### 1.2 另一个子章节
```

**规则**:

- 统一使用一种编号方式
- 保持层级清晰
- 不超过4级

---

### 目录（TOC）规范

**要求**:

- 长文档（>500行）必须有目录
- 目录自动生成或手动维护
- 使用Markdown链接格式

**示例**:

```markdown
## 📑 目录

- [文档标题](#文档标题)
  - [一、章节1](#一章节1)
    - [1.1 子章节](#11-子章节)
  - [二、章节2](#二章节2)
```
