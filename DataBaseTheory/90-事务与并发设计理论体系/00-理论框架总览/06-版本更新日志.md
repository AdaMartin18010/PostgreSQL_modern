## 📝 变更日志格式

### 版本号规则

**主版本号.次版本号.修订号**

- **主版本号**: Phase完成（如1.0, 2.0）
- **次版本号**: 功能新增（如1.1, 1.2）
- **修订号**: Bug修复（如1.1.1）

### 变更类型

- **Added**: 新增功能/文档
- **Changed**: 修改现有内容
- **Deprecated**: 即将废弃
- **Removed**: 删除内容
- **Fixed**: Bug修复
- **Security**: 安全相关

## 🔄 版本回滚策略

### 回滚条件

**需要回滚的情况**:

1. **严重错误**: 理论错误、公式错误
2. **兼容性破坏**: 文档结构大幅变更
3. **质量问题**: 内容质量显著下降

### 回滚流程

```bash
# 1. 创建回滚分支
git checkout -b hotfix/rollback-v1.0.0

# 2. 回滚到上一版本
git revert <commit-hash>

# 3. 修复问题
# 4. 重新发布
git tag v1.0.1
```
