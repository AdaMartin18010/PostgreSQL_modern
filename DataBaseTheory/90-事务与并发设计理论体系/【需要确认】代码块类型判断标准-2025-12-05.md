**特征**: 使用 `├─`, `└─`, `│` 等符号

**示例**:

```
├─ 问题: 如何统一理解并发控制问题？
├─ 理论: 分层状态演化理论
└─ 方法: LSEM统一框架
```

**问题**: 这种应该用 `text` 吗？

---

### 类型2: ASCII框线图

**特征**: 使用 `┌─┐`, `│`, `└─┘` 等框线符号

**示例**:

```
┌────────────────────────────────────────────┐
│  L0: 存储层                            │
│  事务状态                                 │
└────────────────────────────────────────────┘
```

**问题**: 这种应该用 `text` 吗？

---

### 类型3: 文本描述（无特殊符号）

**特征**: 纯文本描述，可能是状态转换、流程说明等

**示例**:

```
初始状态: Tuple(xmin=50, xmax=0, data='A')
    ↓ UPDATE (TxID=100)
旧版本:   Tuple(xmin=50, xmax=100, data='A')
```

**问题**: 这种应该用 `text` 吗？

---

**特征**: 类似代码但可能不是完整可执行的代码

**问题**: 应该如何判断？用具体语言还是 `text`？
