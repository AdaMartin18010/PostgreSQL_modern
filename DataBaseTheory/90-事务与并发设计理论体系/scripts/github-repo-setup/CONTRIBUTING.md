# 贡献指南

感谢您对PostgreSQL事务与并发设计理论体系的关注！我们欢迎所有形式的贡献。

---

## 🤝 如何贡献

### 报告问题

如果您发现了问题或有改进建议，请：

1. 检查 [Issues](https://github.com/your-org/postgresql-concurrency-theory/issues) 中是否已有相关问题
2. 如果没有，请创建新的 Issue，包含：
   - 问题描述
   - 复现步骤
   - 预期行为
   - 实际行为
   - 环境信息

### 提交代码

1. Fork 本仓库
2. 创建特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 创建 Pull Request

### 文档贡献

文档贡献同样重要！您可以：

- 修正错别字和语法错误
- 改进文档结构和可读性
- 添加示例和代码片段
- 翻译文档到其他语言

---

## 📝 代码规范

### Markdown文档

- 使用中文编写
- 遵循现有的文档结构
- 添加适当的代码块和示例
- 保持行长度在100字符以内

### 代码

- 遵循语言特定的代码规范
- 添加必要的注释
- 编写单元测试
- 确保代码可运行

---

- [ ] 更新了相关文档
