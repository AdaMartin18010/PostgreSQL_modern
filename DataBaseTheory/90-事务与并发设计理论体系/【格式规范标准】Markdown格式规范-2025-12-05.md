# Markdown格式规范标准

> **创建日期**: 2025-12-05
> **适用范围**: 本项目所有Markdown文档
> **目标**: 统一格式，提升可读性和维护性

---

## 📋 格式规范总则

### 基本原则

1. **一致性优先** - 所有文档遵循统一格式
2. **可读性优先** - 格式应提升而非降低可读性
3. **可维护性优先** - 格式应便于后续维护和修改

---

## 📐 具体格式规范

### 1. 文档头部格式

**标准模板**:

```markdown
# [编号] | 文档标题

> **模块定位**: 一句话说明定位
> **创建日期**: YYYY-MM-DD
> **文档版本**: x.y.z

---

## 📑 目录

[目录内容]

---
```

**要求**:

- 一级标题使用 `#`，且只有一个
- 使用统一的元数据格式
- 日期格式：YYYY-MM-DD
- 版本格式：x.y.z

---

### 2. 标题层级规范

**标准层级**:

```markdown
# 一级标题 (文档标题，只有一个)

## 二级标题 (主要章节)

### 三级标题 (子章节)

#### 四级标题 (细节，尽量少用)
```

**规则**:

- 标题前后空行
- 层级不超过4级
- 标题简洁明确（<20字）

---

### 3. 代码块格式规范

**标准格式**:

```markdown
    ```语言标识
    代码内容
    ```

```

**语言标识规则**:

| 内容类型 | 语言标识 | 示例 |
|---------|---------|------|
| SQL代码 | `sql` | PostgreSQL查询 |
| Rust代码 | `rust` | Rust实现 |
| Python代码 | `python` | Python脚本 |
| Shell脚本 | `bash` 或 `shell` | 命令行 |
| 配置文件 | `yaml`, `toml`, `json` | 配置文件 |
| 文本/伪代码 | `text` | 文本描述 |
| Mermaid图表 | `mermaid` | 流程图 |
| 数学公式 | LaTeX | 使用 `$$` |

**要求**:

- **所有代码块必须指定语言**
- 代码块前后各空一行
- 代码内容正确缩进
- 长代码块添加注释

**错误示例**:

```markdown
    ❌ 错误：
    ```

    code here

    ```

    ✅ 正确：
    ```python
    code here
    ```

```

---

### 4. 表格格式规范

**标准格式**:

```markdown
    | 列1 | 列2 | 列3 |
    |-----|-----|-----|
    | 内容1 | 内容2 | 内容3 |
    | 内容4 | 内容5 | 内容6 |
```

**要求**:

- 表头与内容对齐
- 表格前后各空一行
- 列宽适中（建议<30字符）
- 内容简洁清晰

---

### 5. 列表格式规范

**无序列表**:

```markdown
- 列表项1
- 列表项2
  - 嵌套项1
  - 嵌套项2
```

**有序列表**:

```markdown
1. 列表项1
2. 列表项2
   1. 嵌套项1
   2. 嵌套项2
```

**要求**:

- 列表前后空行
- 统一使用 `-` 或 `*`（不要混用）
- 嵌套列表统一缩进（2空格或4空格，保持一致）

---

### 6. 链接格式规范

**相对链接**:

```markdown
[链接文本](./相对路径/文件名.md)
```

**外部链接**:

```markdown
[链接文本](https://example.com)
```

**要求**:

- 使用相对路径（相对于当前文档）
- 路径大小写敏感
- 链接文本清晰描述目标
- 定期检查链接有效性

---

### 7. 引用格式规范

**块引用**:

```markdown
> 引用内容
> 可以多行
```

**行内代码**:

```markdown
使用 `代码` 格式
```

**要求**:

- 引用前后空行
- 行内代码使用反引号

---

### 8. 数学公式格式规范

**行内公式**:

```markdown
使用 $公式$ 格式
```

**块级公式**:

```markdown
$$
公式内容
$$
```

**要求**:

- 使用标准的LaTeX语法
- 公式前后空行（块级公式）
- 复杂公式添加说明

---

### 9. 目录（TOC）规范

**标准格式**:

```markdown
## 📑 目录

- [文档标题](#文档标题)
  - [一、章节1](#一章节1)
    - [1.1 子章节](#11-子章节)
  - [二、章节2](#二章节2)
```

**要求**:

- 长文档（>500行）必须有目录
- 目录层级清晰
- 链接锚点正确（Markdown自动生成）

---

### 10. 章节编号规范

**推荐方式1: 中文编号**

```markdown
    ## 一、大章节

    ### 1.1 子章节

    #### 1.1.1 细节
```

**推荐方式2: 数字编号**

```markdown
    ## 1. 大章节

    ### 1.1 子章节

    #### 1.1.1 细节
```

**要求**:

- 统一使用一种编号方式
- 保持层级清晰
- 不超过4级

---

## ❌ 常见格式错误

### 错误1: 代码块无语言标识

```markdown
    ❌ 错误：
    ```

    code here

    ```

    ✅ 正确：

    ```python
    code here
    ```

```

### 错误2: 标题层级混乱

```markdown
    ❌ 错误：
    # 标题1
    ### 标题3（跳过了2）
    ## 标题2

    ✅ 正确：
    # 标题1
    ## 标题2
    ### 标题3
```

### 错误3: 列表格式不一致

```markdown
    ❌ 错误：
    - 项1
    * 项2
    - 项3

    ✅ 正确：
    - 项1
    - 项2
    - 项3
```

### 错误4: 表格不对齐

```markdown
    ❌ 错误：
    | 列1|列2|
    |--|--|
    |内容|内容|

    ✅ 正确：
    | 列1 | 列2 |
    |-----|-----|
    | 内容 | 内容 |
```
