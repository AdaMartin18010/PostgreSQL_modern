## 🔤 MVCC相关缩写

| 缩写 | 全称 | 中文 | 说明 |
|------|------|------|------|
| **MVCC** | Multi-Version Concurrency Control | 多版本并发控制 | PostgreSQL的核心并发控制机制 |
| **HOT** | Heap-Only Tuple | 仅堆元组 | 在同一页内更新元组，不更新索引 |
| **XID** | Transaction ID | 事务ID | 唯一标识事务的32位整数 |
| **SI** | Snapshot Isolation | 快照隔离 | 基于快照的隔离级别 |
| **SSI** | Serializable Snapshot Isolation | 串行化快照隔离 | 实现SERIALIZABLE隔离级别的机制 |

## 🔤 ACID相关缩写

| 缩写 | 全称 | 中文 | 说明 |
|------|------|------|------|
| **ACID** | Atomicity, Consistency, Isolation, Durability | 原子性、一致性、隔离性、持久性 | 数据库事务的四个基本特性 |
| **WAL** | Write-Ahead Logging | 预写日志 | 在修改数据前先写入日志 |
| **RC** | Read Committed | 读已提交 | 只能读取已提交的数据 |
| **RR** | Repeatable Read | 可重复读 | 同一事务内多次读取同一数据，结果一致 |
| **SER** | Serializable | 可串行化 | 最高隔离级别 |
| **2PC** | Two-Phase Commit | 两阶段提交 | 分布式事务提交协议 |
| **CLOG** | Commit Log | 提交日志 | 记录事务提交状态的日志 |

## 🔤 CAP相关缩写

| 缩写 | 全称 | 中文 | 说明 |
|------|------|------|------|
| **CAP** | Consistency, Availability, Partition Tolerance | 一致性、可用性、分区容错性 | CAP定理的三个维度 |
| **CP** | Consistency + Partition Tolerance | 一致性+分区容错性 | 选择一致性和分区容错性，牺牲可用性 |
| **AP** | Availability + Partition Tolerance | 可用性+分区容错性 | 选择可用性和分区容错性，牺牲一致性 |
| **CA** | Consistency + Availability | 一致性+可用性 | 选择一致性和可用性，牺牲分区容错性 |
| **BASE** | Basically Available, Soft state, Eventual consistency | 基本可用、软状态、最终一致性 | AP模式的实现方式 |
