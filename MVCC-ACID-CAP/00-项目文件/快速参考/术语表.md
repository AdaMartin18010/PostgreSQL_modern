## 📚 MVCC术语

| 术语 | 英文 | 定义 | 关联 |
|------|------|------|------|
| **多版本并发控制** | Multi-Version Concurrency Control (MVCC) | 通过维护多个版本实现并发控制的机制 | ACID、快照隔离 |
| **快照** | Snapshot | 事务在某个时间点的数据视图 | 隔离性、可见性 |
| **快照隔离** | Snapshot Isolation | 基于快照的隔离级别 | REPEATABLE READ |
| **版本链** | Version Chain | 同一逻辑元组的多个物理版本形成的链表 | 元组、版本 |
| **元组** | Tuple | 数据库表中的一行数据 | 版本、可见性 |
| **活元组** | Live Tuple | 当前可见的元组版本 | 死元组、可见性 |
| **死元组** | Dead Tuple | 已被删除或更新的旧版本元组 | 活元组、VACUUM |
| **事务ID** | Transaction ID (XID) | 唯一标识事务的32位整数 | t_xmin、t_xmax |
| **t_xmin** | Transaction Minimum XID | 创建元组版本的事务XID | 可见性、版本 |
| **t_xmax** | Transaction Maximum XID | 删除或更新元组版本的事务XID | 可见性、版本 |
| **可见性** | Visibility | 元组版本对事务是否可见 | 快照、隔离性 |
| **清理** | VACUUM | 清理死元组，回收存储空间 | 死元组、表膨胀 |
| **填充因子** | Fillfactor | 表页中预留的空间比例 | HOT更新、版本链 |
| **仅堆元组更新** | Heap-Only Tuple (HOT) Update | 在同一页内更新元组，不更新索引 | 填充因子、性能 |

## 📚 ACID术语

| 术语 | 英文 | 定义 | 关联 |
|------|------|------|------|
| **原子性** | Atomicity | 事务的所有操作要么全部成功，要么全部失败 | 一致性、事务 |
| **一致性** | Consistency | 事务执行前后，数据库保持一致状态 | 原子性、隔离性 |
| **隔离性** | Isolation | 并发事务之间相互隔离，互不干扰 | 快照隔离、隔离级别 |
| **持久性** | Durability | 已提交事务的修改永久保存 | WAL、提交 |
| **事务** | Transaction | 一组数据库操作的逻辑单元 | ACID、并发 |
| **提交** | Commit | 事务的最终确认操作 | 持久性、WAL |
| **回滚** | Rollback | 撤销事务的所有修改 | 原子性、一致性 |
| **中止** | Abort | 事务的回滚操作 | 回滚、原子性 |
| **隔离级别** | Isolation Level | 定义事务隔离程度的级别 | 隔离性、快照隔离 |
| **读已提交** | Read Committed | 只能读取已提交的数据 | 隔离级别、默认 |
| **可重复读** | Repeatable Read | 同一事务内多次读取同一数据，结果一致 | 隔离级别、快照隔离 |
| **可串行化** | Serializable | 最高隔离级别，保证事务执行结果等价于某个串行执行 | 隔离级别、SSI |
| **预写日志** | Write-Ahead Logging (WAL) | 在修改数据前先写入日志 | 持久性、提交 |
| **串行化快照隔离** | Serializable Snapshot Isolation (SSI) | 实现SERIALIZABLE隔离级别的机制 | 可串行化、冲突检测 |

## 📚 CAP术语

| 术语 | 英文 | 定义 | 关联 |
|------|------|------|------|
| **一致性** | Consistency | 所有节点在同一时间看到相同的数据 | 可用性、分区容错性 |
| **可用性** | Availability | 系统在合理时间内响应请求的能力 | 一致性、分区容错性 |
| **分区容错性** | Partition Tolerance | 系统在网络分区情况下继续工作的能力 | 一致性、可用性 |
| **CAP定理** | CAP Theorem | 在分布式系统中，一致性、可用性、分区容错性三者不能同时满足 | 分布式系统、权衡 |
| **CP模式** | CP Mode | 选择一致性和分区容错性，牺牲可用性 | CAP定理、PostgreSQL |
| **AP模式** | AP Mode | 选择可用性和分区容错性，牺牲一致性 | CAP定理、最终一致性 |
| **CA模式** | CA Mode | 选择一致性和可用性，牺牲分区容错性 | CAP定理、单机系统 |

## 📚 通用术语

| 术语 | 英文 | 定义 | 关联 |
|------|------|------|------|
| **并发控制** | Concurrency Control | 控制并发事务访问共享资源的机制 | MVCC、ACID |
| **锁** | Lock | 控制资源访问的机制 | 并发控制、隔离性 |
| **死锁** | Deadlock | 两个或多个事务相互等待对方释放锁 | 并发控制、锁 |
| **表膨胀** | Table Bloat | 表中死元组过多导致表大小增加 | 死元组、VACUUM |
| **检查点** | Checkpoint | 将内存中的脏页写入磁盘的操作 | WAL、持久性 |
| **流复制** | Streaming Replication | 实时复制WAL到备用服务器 | 高可用、CAP |
| **逻辑复制** | Logical Replication | 基于逻辑变更的复制方式 | 高可用、CAP |
| **故障转移** | Failover | 主服务器故障时切换到备用服务器 | 高可用、可用性 |
