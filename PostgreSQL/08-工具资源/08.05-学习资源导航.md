## 一、概述

### 1.1 资源导航目标

学习资源导航旨在为PostgreSQL学习者提供完整的学习路径和资源索引，帮助不同水平的用户找到合适的学习资源。

**核心目标**：

- **学习路径**：提供清晰的学习路径规划
- **资源索引**：整理各类学习资源
- **快速定位**：帮助用户快速找到所需资源
- **持续更新**：保持资源的时效性

### 1.2 学习路径

**初学者路径**（1-2周）：

1. 阅读官方文档快速入门
2. 完成基础SQL练习
3. 搭建本地环境实践

**进阶路径**（1-3个月）：

1. 深入学习核心概念
2. 阅读相关书籍
3. 参与社区讨论
4. 完成实战项目

**专家路径**（6个月+）：

1. 阅读源码和论文
2. 参与开源贡献
3. 参加技术会议
4. 分享实践经验

### 1.3 资源分类

- **官方资源**：官方文档、源码、博客
- **社区资源**：邮件列表、论坛、会议
- **扩展生态**：扩展、工具、云服务
- **书籍资源**：入门、进阶、专业书籍
- **学术论文**：核心论文、研究论文
- **实验工具**：基准测试、实验仓库

   - 作者：Michael Stonebraker, et al.

## 七、实验与基准

### 7.1 基准测试工具

**pgbench**：

- **官方文档**：<https://www.postgresql.org/docs/current/pgbench.html>
- **功能**：PostgreSQL内置基准测试工具
- **用途**：性能测试、压力测试

**使用示例**：

```bash
# 初始化测试数据
pgbench -i -s 100 mydb

# 运行基准测试
pgbench -c 10 -j 2 -T 60 mydb

# 自定义测试脚本
pgbench -f custom_script.sql -c 10 -T 60 mydb
```

**TPC基准**：

- **TPC-C**：<https://www.tpc.org/tpcc/>
  - 用途：OLTP工作负载测试
  - 应用：事务处理性能测试

- **TPC-H**：<https://www.tpc.org/tpch/>
  - 用途：决策支持工作负载测试
  - 应用：分析查询性能测试

## 十、学习建议

### 10.1 初学者建议

1. **从官方文档开始**：阅读官方文档快速入门章节
2. **动手实践**：搭建本地环境，完成基础练习
3. **参与社区**：加入邮件列表，提问和讨论
4. **阅读书籍**：选择一本入门书籍系统学习

1. **深入学习核心概念**：事务、并发控制、查询优化
