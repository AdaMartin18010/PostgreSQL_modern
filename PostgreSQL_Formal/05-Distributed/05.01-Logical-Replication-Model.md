## 1. 概念定义

### 1.1 复制模型

$$
\text{Replication} := \langle P, L, A, C \rangle
$$

其中:

- $P$: 发布者 (Publisher)
- $L$: 逻辑解码
- $A$: 应用者 (Apply)
- $C$: 复制一致性

### 1.2 复制类型

| 类型 | 粒度 | 延迟 | 冲突处理 |
|------|------|------|----------|
| **物理复制** | 块级 | 低 | 无 (主从完全一致) |
| **逻辑复制** | 行级 | 中 | 应用层处理 |
| **双向复制** | 表级 | 高 | 冲突检测 |

## 2. 一致性模型

### 2.1 复制延迟

$$
\text{Replication Lag} = T_{commit}^{primary} - T_{visible}^{replica}
$$

**监控**:

```sql
-- 查看复制延迟
SELECT
    client_addr,
    backend_start,
    sent_lsn,
    write_lsn,
    flush_lsn,
    replay_lsn
FROM pg_stat_replication;
```

### 2.2 一致性级别

| 级别 | 保证 | PostgreSQL支持 |
|------|------|----------------|
| **最终一致性** | 延迟后一致 | ✅ 逻辑复制 |
| **因果一致性** | 因果关系保持 | ⚠️ 部分支持 |
| **顺序一致性** | 全局顺序 | ❌ 不支持 |
| **强一致性** | 实时一致 | ✅ 同步复制 |

## 3. 冲突处理

### 3.1 冲突类型

**主键冲突**:
$$
T_1: INSERT (id=1) \quad \text{vs} \quad T_2: INSERT (id=1)
$$

**更新丢失**:
$$
T_1: UPDATE x=1 \quad \text{vs} \quad T_2: UPDATE x=2
$$

### 3.2 解决策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| **Last-Write-Wins** | 时间戳大的胜出 | 简单覆盖 |
| **First-Write-Wins** | 先写入的保留 | 防止覆盖 |
| **Merge** | 合并冲突值 | 复杂数据结构 |
| **Abort** | 中止事务 | 严格一致性 |

## 5. 反例分析

### 5.1 常见误用

**反例 1: DDL复制**

```sql
-- 逻辑复制不复制DDL
ALTER TABLE users ADD COLUMN age INT;
-- 订阅端不会自动添加列

-- 解决方案: 手动在订阅端执行DDL
```

**反例 2: 大事务延迟**

```sql
-- 大事务导致复制延迟
BEGIN;
INSERT INTO logs SELECT * FROM huge_table;
COMMIT;

-- 解决方案: 分批提交
```
