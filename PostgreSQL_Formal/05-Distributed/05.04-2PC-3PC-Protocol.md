# 2PC vs 3PC 形式化

> **文档类型**: 分布式系统
> **创建日期**: 2026-03-04

---

## 1. 两阶段提交 (2PC)

### 1.1 协议

```
Phase 1 (Prepare):
    Coordinator → Prepare → Participants
    Coordinator ← Yes/No ← Participants

Phase 2 (Commit/Abort):
    Coordinator → Commit/Abort → Participants
```

### 1.2 问题

**阻塞问题**: 如果Coordinator在Prepare后崩溃，Participants必须等待。

---

## 2. 三阶段提交 (3PC)

### 2.1 协议

```
Phase 1 (CanCommit):
    Coordinator → CanCommit? → Participants
    Coordinator ← Yes/No ← Participants

Phase 2 (PreCommit):
    Coordinator → PreCommit → Participants
    Coordinator ← ACK ← Participants

Phase 3 (DoCommit):
    Coordinator → DoCommit → Participants
```

### 2.2 优势

非阻塞: 超时后可以独立决定。

---

## 3. 对比

| 特性 | 2PC | 3PC |
|------|-----|-----|
| 阶段数 | 2 | 3 |
| 阻塞 | 是 | 否 |
| 网络开销 | 低 | 高 |
| 实现复杂度 | 低 | 高 |
| 使用场景 | 通用 | 高可用要求 |
