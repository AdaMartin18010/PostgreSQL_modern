## 1. Raft共识算法

### 1.1 角色

| 角色 | 职责 |
|------|------|
| **Leader** | 处理所有客户端请求 |
| **Follower** | 复制Leader日志 |
| **Candidate** | 选举期间的角色 |

### 1.2 安全保证

- **选举安全**: 最多一个Leader在同一任期
- **日志匹配**: 相同索引的日志条目相同
- **Leader完备**: Leader包含所有已提交条目

## 2. Patroni架构

```
┌─────────┐     ┌─────────┐     ┌─────────┐
│ Primary │←───→│ Replica │←───→│ Replica │
│ (Leader)│     │         │     │         │
└────┬────┘     └─────────┘     └─────────┘
     │
     ↓ Replication
┌─────────┐
│  DCS    │ (etcd/ZooKeeper)
└─────────┘
```

## 3. 故障转移

### 3.1 自动故障检测

```
Leader心跳超时 → 开始选举 → 新Leader产生 → 更新DCS
```
