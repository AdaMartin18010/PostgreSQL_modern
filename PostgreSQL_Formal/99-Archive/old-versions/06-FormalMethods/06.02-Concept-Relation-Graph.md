## 1. 概念图谱结构

### 1.1 节点定义

每个概念是一个节点:

$$
\text{Node} := \langle id, name, type, definition \rangle
$$

### 1.2 边定义

概念之间的关系:

$$
\text{Edge} := \langle source, target, relation_type \rangle
$$

关系类型:

- **ISA**: 泛化关系 (A是B的一种)
- **PART-OF**: 组成关系 (A是B的一部分)
- **DEPENDS-ON**: 依赖关系 (A依赖B)
- **IMPLEMENTS**: 实现关系 (A实现B)
- **CONTRASTS-WITH**: 对比关系 (A与B对比)

## 2. 核心概念图谱

### 2.1 事务与并发

```
Transaction
    ├── ISA: ACID
    │       ├── Atomicity
    │       ├── Consistency
    │       ├── Isolation
    │       └── Durability
    ├── DEPENDS-ON: Concurrency Control
    │       ├── ISA: 2PL
    │       ├── ISA: MVCC
    │       │       └── IMPLEMENTS: PostgreSQL
    │       └── ISA: SSI
    └── DEPENDS-ON: Recovery
            └── IMPLEMENTS: WAL
                    └── PART-OF: ARIES
```

### 2.2 存储引擎

```
Storage Engine
    ├── PART-OF: Buffer Pool
    │       └── IMPLEMENTS: Clock Algorithm
    ├── PART-OF: Index Manager
    │       ├── ISA: BTree
    │       ├── ISA: Hash
    │       └── ISA: GIN
    ├── PART-OF: WAL Manager
    │       └── DEPENDS-ON: Disk I/O
    │               ├── ISA: Sync I/O
    │               └── ISA: AIO
    └── PART-OF: Heap Access Method
```
