## 1. 概念定义

### 1.1 代价模型

$$
\text{Cost}(Plan) = \text{IO Cost} + \text{CPU Cost} + \text{Network Cost}
$$

### 1.2 基本假设

| 假设 | 符号 | 说明 |
|------|------|------|
| **选择率** | $sel$ | 满足条件的元组比例 |
| **基数** | $card$ | 元组数量 |
| **页大小** | $B$ | 每页字节数 |
| **元组大小** | $t$ | 每元组字节数 |

## 2. 代价公式

### 2.1 顺序扫描

$$
C_{seq} = N_{pages} \times \text{seq_page_cost} + N_{tuples} \times \text{cpu_tuple_cost}
$$

### 2.2 索引扫描

$$
C_{index} = N_{index_pages} \times \text{random_page_cost} + N_{tuples} \times \text{cpu_index_tuple_cost}
$$

### 2.3 连接代价

**Nested Loop**:
$$
C_{NL} = |R| + |R| \times |S| \times \text{join_selectivity}
$$

**Hash Join**:
$$
C_{HJ} = 2 \times |R| + |S| \quad \text{(构建哈希表 + 探测)}
$$

**Merge Join**:
$$
C_{MJ} = sort(R) + sort(S) + |R| + |S|
$$

## 3. 统计信息

### 3.1 直方图

PostgreSQL使用MCV (Most Common Values) 和直方图:

```
列值分布:
[min] |----|----|----|----| [max]
       ↑频率↑
```

### 3.2 选择率估计

**等值条件**:
$$sel(A = v) = \frac{1}{ndistinct(A)}$$

**范围条件**:
$$sel(A \le v) = \frac{v - min(A)}{max(A) - min(A)}$$

## 4. PostgreSQL实现

### 4.1 代价参数

```ini
# postgresql.conf
seq_page_cost = 1.0          # 顺序页读取代价
random_page_cost = 4.0       # 随机页读取代价
cpu_tuple_cost = 0.01        # CPU处理每个元组代价
cpu_index_tuple_cost = 0.005 # CPU处理索引元组代价
cpu_operator_cost = 0.0025   # CPU执行操作代价
parallel_tuple_cost = 0.1    # 并行处理代价
```

### 4.2 统计信息收集

```sql
-- 收集统计信息
ANALYZE users;

-- 查看统计信息
SELECT * FROM pg_stats WHERE tablename = 'users';
```

## 5. 反例分析

### 5.1 常见误用

**反例 1: 统计信息过时**

```sql
-- 表数据大幅变化后未ANALYZE
-- 优化器基于错误统计选择计划

-- 解决方案
ANALYZE large_table;
```

**反例 2: 代价参数不适应存储**

```ini
# SSD上仍使用HDD参数
random_page_cost = 4.0  # SSD应设为1.1-2.0

# 正确配置(SSD)
random_page_cost = 1.1
```
