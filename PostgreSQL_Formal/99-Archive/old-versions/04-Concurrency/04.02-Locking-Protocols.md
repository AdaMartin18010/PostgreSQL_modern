## 1. 概念定义

### 1.1 锁类型

PostgreSQL实现多种锁粒度:

| 锁类型 | 粒度 | 冲突关系 |
|--------|------|----------|
| **表级锁** | Relation | AccessShareLock, RowExclusiveLock, etc. |
| **行级锁** | Tuple | FOR UPDATE, FOR SHARE, etc. |
| **页级锁** | Page | 内部使用 |
| **咨询锁** | 应用定义 | 用户自定义 |

### 1.2 锁模式矩阵

```text
          S   X   IS  IX
        ┌───┬───┬───┬───┐
    S   │ ✓ │ ✗ │ ✓ │ ✗ │
        ├───┼───┼───┼───┤
    X   │ ✗ │ ✗ │ ✗ │ ✗ │
        ├───┼───┼───┼───┤
    IS  │ ✓ │ ✗ │ ✓ │ ✓ │
        ├───┼───┼───┼───┤
    IX  │ ✗ │ ✗ │ ✓ │ ✓ │
        └───┴───┴───┴───┘
```

## 2. 两阶段锁 (2PL)

### 2.1 定义

**定义 2.1 (两阶段锁)**:
事务遵循2PL如果:

1. **增长阶段**: 只能获取锁，不能释放
2. **收缩阶段**: 只能释放锁，不能获取

### 2.2 严格两阶段锁 (Strict 2PL)

锁直到事务结束才释放:
$$
\forall l \in \text{Locks}(T): \text{unlock}(l) \prec \text{commit}(T)
$$

**PostgreSQL实现**:

- 行锁保持到事务结束
- 表锁根据类型决定保持时间

## 3. 死锁检测

### 3.1 等待图 (Wait-For Graph)

$$
WFG = (V, E) \text{ 其中 } V = \{T_i\}, E = \{(T_i, T_j) \mid T_i \text{ waits for } T_j\}

## 4. 锁升级与粒度

### 4.1 意向锁 (Intention Locks)

**定义 4.1 (意向锁)**:

- IS (Intention Shared): 意向共享锁
- IX (Intention Exclusive): 意向排他锁

**作用**: 支持多粒度锁定，避免检查所有子节点。

### 4.2 锁升级

当行锁数量超过阈值时，升级为表锁:
$$
\text{UpgradeCondition} := |\text{RowLocks}(T)| > \text{max_locks_per_transaction}
$$
