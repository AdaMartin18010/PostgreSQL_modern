## 1. 概念定义

### 1.1 Buffer Pool数学模型

$$
\text{BufferPool} := \langle N, F, P, R \rangle
$$

其中:

- $N$: Buffer pool大小 (页数)
- $F$: 空闲页集合, $F \subseteq \{1, ..., N\}$
- $P$: 已使用页集合, $P: \text{PageId} \rightarrow \text{Frame}
- $R$: 替换策略

### 1.2 页状态

$$
\text{PageState} := \{\text{FREE}, \text{CLEAN}, \text{DIRTY}, \text{PINNED}\}
$$

## 2. 替换策略

### 2.1 Clock算法 (PostgreSQL实现)

**算法描述**:

```
ClockSweep():
    while true:
        frame = buffer_pool[clock_hand]
        if frame.ref_count == 0:
            if frame.usage_count > 0:
                frame.usage_count--
            else:
                return frame  // 找到受害者
        clock_hand = (clock_hand + 1) % N
```

**形式化**:
$$
\text{Victim} := \min_{i} \{i \mid ref[i] = 0 \land usage[i] = 0\}
$$

### 2.2 命中率分析

**命中概率**:
$$
P_{hit} = \frac{|\text{WorkingSet}|}{N}
$$

其中Working Set是活跃页集合。

## 3. PostgreSQL实现

### 3.1 数据结构

```c
// src/include/storage/bufmgr.h

typedef struct BufferDesc {
    BufferTag   tag;            /* 页标识 */
    BufFlags    flags;          /* 状态标志 */
    uint16      usage_count;    /* Clock算法使用计数 */
    unsigned    ref_count;      /* 引用计数 */
    slock_t     buf_hdr_lock;   /* 保护锁 */
} BufferDesc;
```

### 3.2 缓冲命中统计

```sql
-- 查看Buffer Pool命中率
SELECT
    datname,
    blks_hit,
    blks_read,
    blks_hit::float / (blks_hit + blks_read) AS hit_ratio
FROM pg_stat_database;
```

## 4. 反例分析

### 4.1 常见误用

**反例 1: Buffer Pool过小**

```ini
shared_buffers = 128MB  # 对于大表太小

-- 结果: 频繁的页换入换出，性能下降
```

**反例 2: 忽略清理**

```sql
-- 大量UPDATE导致脏页过多
-- 检查点压力增大
-- 响应时间抖动

-- 解决方案
ALTER SYSTEM SET checkpoint_completion_target = 0.9;
```
