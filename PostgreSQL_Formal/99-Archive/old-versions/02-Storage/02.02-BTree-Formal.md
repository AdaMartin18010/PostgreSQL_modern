## 1. 概念定义

### 1.1 B+Tree定义

B+Tree是一个$n$叉搜索树，满足:

1. 每个节点最多 $2d-1$ 个键，最少 $d-1$ 个键 (根节点除外)
2. 所有叶节点在同一层
3. 叶节点通过指针链接

$$
\text{B+Tree}(d) := \langle r, m, h, L \rangle
$$

其中:

- $d$: 最小度数
- $r$: 根节点
- $m$: 最大键数
- $h$: 树高
- $L$: 叶节点链表

### 1.2 节点结构

**内部节点**:
$$
\text{InternalNode} := \langle P_0, K_1, P_1, K_2, ..., K_n, P_n \rangle
$$

其中 $P_i$ 是指向子节点的指针，$K_i$ 是键值。

**叶节点**:
$$
\text{LeafNode} := \langle (K_1, R_1), (K_2, R_2), ..., (K_n, R_n), P_{next} \rangle
$$

其中 $R_i$ 是记录指针(TID)，$P_{next}$ 指向下一个叶节点。

## 2. 操作复杂度

### 2.1 时间复杂度

| 操作 | 平均 | 最坏 | 说明 |
|------|------|------|------|
| **搜索** | $O(\log_d N)$ | $O(\log_d N)$ | 树高 |
| **插入** | $O(\log_d N)$ | $O(\log_d N)$ | 可能分裂 |
| **删除** | $O(\log_d N)$ | $O(\log_d N)$ | 可能合并 |
| **范围查询** | $O(\log_d N + M)$ | $O(\log_d N + M)$ | $M$为结果数 |

其中 $N$ 是记录数，$d$ 是最小度数。

### 2.2 空间复杂度

$$
\text{Space} = O(N \cdot (\text{key_size} + \text{ptr_size}))
$$

填充因子通常 $
ho \approx 0.67$。

## 3. 并发控制

### 3.1 B-Link Tree

PostgreSQL使用B-Link Tree支持高并发:

```
兄弟节点指针:
┌─────────┐    ┌─────────┐    ┌─────────┐
│ Leaf 1  │───→│ Leaf 2  │───→│ Leaf 3  │
└─────────┘    └─────────┘    └─────────┘
```

**优势**: 分裂时只需锁定两个节点，而非整个子树。

### 3.2 悲观锁 vs 乐观锁

| 策略 | 读操作 | 写操作 | 适用场景 |
|------|--------|--------|----------|
| **悲观锁** | 共享锁 | 排他锁 | 高冲突 |
| **乐观锁** | 无锁 | 验证后提交 | 低冲突 |

## 4. PostgreSQL实现

### 4.1 页结构

```c
// src/include/access/nbtree.h

/* B-tree页头 */
typedef struct BTPageOpaqueData {
    BlockNumber btpo_prev;      /* 左兄弟 */
    BlockNumber btpo_next;      /* 右兄弟 */
    union {
        BlockNumber btpo_parent;    /* 父节点 */
        uint32      btpo_flags;     /* 标志 */
    } btpo;
    uint16      btpo_level;     /* 树层级(叶节点=0) */
    IndexTupleData btpo_tup;    /* 特殊元组 */
} BTPageOpaqueData;
```

### 4.2 索引扫描

```sql
-- 等值查询
SELECT * FROM users WHERE id = 100;
-- 复杂度: O(log N)

-- 范围查询
SELECT * FROM users WHERE age BETWEEN 20 AND 30;
-- 复杂度: O(log N + M)
```

## 5. 反例分析

### 5.1 常见误用

**反例 1: 低选择性列索引**

```sql
-- 性别只有两种值
CREATE INDEX idx_gender ON users(gender);
-- 选择性 = 2/N，几乎无用
```

**反例 2: 索引列参与运算**

```sql
-- 无法使用索引
SELECT * FROM users WHERE YEAR(created_at) = 2025;

-- 正确做法
SELECT * FROM users
WHERE created_at >= '2025-01-01'
  AND created_at < '2026-01-01';
```
