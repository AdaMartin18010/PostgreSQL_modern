# 分布式事务形式化

> **文档类型**: 核心理论
> **创建日期**: 2026-03-04

---

## 1. 概念定义

### 1.1 分布式事务

跨越多个节点的事务:

$$
T = \{T_1, T_2, ..., T_n\}, \quad T_i \text{ on node } i
$$

### 1.2 ACID挑战

| 性质 | 挑战 | 解决方案 |
|------|------|----------|
| **原子性** | 部分节点提交 | 2PC/3PC |
| **一致性** | 数据复制延迟 | 共识算法 |
| **隔离性** | 全局调度 | 全局锁 |
| **持久性** | 节点故障 | 复制 |

---

## 2. 两阶段提交 (2PC)

### 2.1 协议流程

```
阶段1 (投票):
    Coordinator → Prepare? → Participants
    Coordinator ← Yes/No ← Participants

阶段2 (提交/中止):
    if all Yes:
        Coordinator → Commit → Participants
    else:
        Coordinator → Abort → Participants
```

### 2.2 状态机

Coordinator状态:
$$
\{\text{INIT}, \text{PREPARING}, \text{PREPARED}, \text{COMMITTING}, \text{COMMITTED}, \text{ABORTED}\}
$$

---

## 3. CAP定理

**定理**: 分布式系统无法同时满足:

- **C**onsistency (一致性)
- **A**vailability (可用性)
- **P**artition tolerance (分区容错)

$$
\neg(C \land A \land P)
$$
