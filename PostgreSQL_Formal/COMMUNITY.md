# 社区沟通渠道

欢迎加入 PostgreSQL 现代教程社区！我们提供多种沟通渠道以满足不同的交流需求。

## 官方渠道

### GitHub Discussions 💬

**用途**：技术讨论、问题求助、经验分享

- **链接**：[GitHub Discussions](https://github.com/luyatshimbalanga/PostgreSQL_modern/discussions)
- **适合场景**：
  - 深入的技术讨论
  - 学习方法分享
  - 最佳实践交流
  - 一般性问题咨询
- **响应时间**：通常 24-48 小时内

### GitHub Issues 🐛

**用途**：错误报告、文档改进、特性请求

- **链接**：[GitHub Issues](https://github.com/luyatshimbalanga/PostgreSQL_modern/issues)
- **适合场景**：
  - 报告文档错误
  - 提交改进建议
  - 请求新特性
  - 追踪问题修复进度
- **分类标签**：
  - `bug` - 错误报告
  - `documentation` - 文档改进
  - `feature` - 特性请求
  - `question` - 问题咨询
  - `good first issue` - 适合新手

## 参与指南

### 如何提问

获得高质量回答的提示：

1. **先搜索**：在提问前搜索已有 Issues 和 Discussions
2. **提供上下文**：说明 PostgreSQL 版本、操作系统、具体问题场景
3. **描述尝试**：说明你已经尝试过什么解决方案
4. **保持礼貌**：记住帮助你的人都是志愿者

### 如何贡献

参与社区的多种方式：

- 📝 **改进文档**：修复错误、补充内容、改进表达
- 💻 **分享经验**：在 Discussions 分享你的使用心得
- 🐛 **报告问题**：发现并报告文档中的问题
- 🌟 **推广项目**：向他人推荐本项目
- 💡 **提出想法**：在 Discussions 提出新想法

### 行为准则

参与社区即表示您同意遵守我们的 [行为准则](./CODE_OF_CONDUCT.md)。

核心原则：

- 尊重他人
- 建设性讨论
- 包容多样
- 乐于助人

## 常见问题

**Q: 我有问题应该在哪里提问？**

A: 取决于问题类型：

- 文档错误 → GitHub Issues
- 技术讨论 → GitHub Discussions
- 快速问题 → 等待 Discord/微信群上线

**Q: 我想贡献代码/文档，从哪里开始？**

A: 查看 [贡献指南](./CONTRIBUTING.md)，并寻找带有 `good first issue` 标签的 Issue。

**Q: 如何成为项目维护者？**

A: 通过持续的贡献展示你的能力和承诺，维护者团队会主动邀请活跃的贡献者加入。

**Q: 企业如何参与合作？**

A: 欢迎通过 GitHub Discussions 或 Issue 联系我们，探讨合作可能性。
