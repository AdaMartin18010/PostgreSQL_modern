## 1. 理论基础与依赖模型

### 1.1 事务依赖理论

在并发事务执行中，事务之间通过数据访问产生依赖关系。Adya模型定义了三种基本依赖类型：

**定义 1.1 (读写依赖 WR-Dependency)**:

事务 $T_j$ 读写依赖于 $T_i$ ($T_i \xrightarrow{wr} T_j$) 当且仅当:

$$
\exists x: w_i(x_i) \prec r_j(x_i) \land x_i \text{ is the version written by } T_i
$$

形式化:

$$
T_i \xrightarrow{wr(x)} T_j := w_i(x_i) \prec r_j(x_i) \land \nexists T_k: w_i(x_i) \prec w_k(x_k) \prec r_j(x_i)
$$

**定义 1.2 (写写依赖 WW-Dependency)**:

事务 $T_j$ 写写依赖于 $T_i$ ($T_i \xrightarrow{ww} T_j$) 当且仅当:

$$
\exists x: w_i(x) \prec w_j(x) \land x \text{ is the same data object}
$$

形式化:

$$
T_i \xrightarrow{ww(x)} T_j := w_i(x_i) \prec w_j(x_j) \land x_i = x_j = x
$$

**定义 1.3 (读写反依赖 RW-Antidependency)**:

事务 $T_j$ 读写反依赖于 $T_i$ ($T_i \xrightarrow{rw} T_j$) 当且仅当:

$$
\exists x: r_i(x_k) \prec w_j(x_j) \land T_k \neq T_j \land x_k \text{ is the version read by } T_i
$$

形式化:

$$
T_i \xrightarrow{rw(x)} T_j := r_i(x_k) \prec w_j(x_j) \land T_i \neq T_j \land \nexists T_m: r_i(x_k) \prec w_m(x_m) \prec w_j(x_j)
$$

### 1.2 依赖图模型

**定义 1.4 (依赖图)**:

对于历史 $H$，依赖图 $DG(H) = (V, E)$ 定义如下:

- $V = \{T_i \mid T_i \in \text{trans}(H)\}$
- $E = E_{wr} \cup E_{ww} \cup E_{rw}$
  - $E_{wr} = \{(T_i, T_j) \mid T_i \xrightarrow{wr} T_j\}$
  - $E_{ww} = \{(T_i, T_j) \mid T_i \xrightarrow{ww} T_j\}$
  - $E_{rw} = \{(T_i, T_j) \mid T_i \xrightarrow{rw} T_j\}$

### 1.3 版本序与读取依赖

**定义 1.5 (版本序)**:

对于数据对象 $x$，版本序 $<_x$ 定义为:

$$
x_i <_x x_j \iff w_i(x_i) \prec w_j(x_j)
$$

**定义 1.6 (从属读取)**:

事务 $T_j$ 从属地读取 $T_i$ 对 $x$ 的写入，当且仅当:

$$
T_i \xrightarrow{wr(x)} T_j \land T_i = \max_{<_x}\{T_k \mid w_k(x_k) \prec r_j(x_k)\}
$$

## 2. 直接序列化图 (DSG) 理论

### 2.1 DSG定义

**定义 2.1 (直接序列化图)**:

对于历史 $H$，直接序列化图 $DSG(H) = (V, E)$ 定义为:

$$
V = \{T_i \mid T_i \in \text{trans}(H)\} \cup \{T_0\}
$$

其中 $T_0$ 为虚拟初始事务，写入所有初始值。

$$
E = \{(T_i, T_j) \mid T_i \xrightarrow{wr} T_j \lor T_i \xrightarrow{ww} T_j \lor T_i \xrightarrow{rw} T_j\}
$$

### 2.2 DSG性质

**性质 2.1 (DSG完备性)**:

如果调度 $H$ 是可串行化的，则 $DSG(H)$ 无环。

**性质 2.2 (DSG冲突等价)**:

两个调度冲突等价当且仅当它们的DSG相同。

### 2.3 多版本DSG (MV-DSG)

在多版本系统中，DSG扩展为MV-DSG，考虑版本信息:

$$
MV\text{-}DSG(H) = (V, E_{wr}^{MV}, E_{ww}^{MV}, E_{rw}^{MV})
$$

其中:

$$
E_{wr}^{MV} = \{(T_i, T_j, x, v_i) \mid T_i \xrightarrow{wr(x,v_i)} T_j\}
$$

## 3. 现象定义与分类

### 3.1 异常现象体系

Adya模型定义了层次化的异常现象体系：

#### 3.1.1 G0 - 脏写 (Dirty Write)

**定义 3.1**:

$$
G0 := \exists \text{ cycle in } DSG \text{ consisting of only } ww \text{ edges}
$$

**含义**: 事务覆盖了彼此未提交的写入。

**示例**:

```
T1: w(x) ------------> w(y) ----> commit
        \                 /
         \               /
          w(x) ------> w(y)
T2:                    commit

G0: T1 →ww(x)→ T2 →ww(y)→ T1
```

#### 3.1.2 G1 - 脏读 (Dirty Read)

**定义 3.2**:

$$
G1 := G1a \lor G1b
$$

**G1a - 中止读取 (Aborted Reads)**:

$$
G1a := \exists T_j, T_i: w_i(x) \prec r_j(x) \prec a_i \prec c_j
$$

事务 $T_j$ 读取了中止事务 $T_i$ 的写入。

**G1b - 中间读取 (Intermediate Reads)**:

$$
G1b := \exists T_j, T_i, x: w_i(x_{old}) \prec w_i(x_{new}) \prec r_j(x_{old}) \prec c_j
$$

事务 $T_j$ 读取了 $T_i$ 对 $x$ 的旧版本，但 $T_i$ 后来覆盖了该版本。

#### 3.1.3 G2 - 反依赖环

**定义 3.3**:

$$
G2 := \exists \text{ cycle in } DSG \text{ containing at least one } rw \text{ edge}
$$

**G2-item**: 单数据对象的反依赖环

**G2**: 多数据对象的反依赖环 (不可串行化)

#### 3.1.4 G-SI - 快照隔离异常

**定义 3.4 (快照隔离)**:

快照隔离保证:

1. 事务读取数据库的一个一致快照 (快照时间戳)
2. 更新的事务在提交时检查写冲突

**G-SI现象**:

$$
G\text{-}SI := \exists T_i, T_j, x:
    T_i \xrightarrow{ww(x)} T_j \land
    \neg(T_i \xrightarrow{wr} T_j) \land
    \text{snapshot}(T_j) < \text{commit}(T_i)
$$

即：事务 $T_j$ 覆盖了 $T_i$ 的写入，但 $T_j$ 没有读到 $T_i$ 的写入。

### 3.2 现象层次结构

```
异常现象层次:

G0 (脏写)
 ├── 禁止 G0 → PL-1
 │
 ├── G1a (中止读取)
 ├── G1b (中间读取)
 │   └── 禁止 G0, G1 → PL-2
 │
 ├── G-SI (快照隔离异常)
 │   └── 禁止 G0, G1, G-SI → PL-2.99
 │
 ├── G2-item (单对象反依赖)
 └── G2 (反依赖环)
     └── 禁止 G0, G1, G2 → PL-3
```

## 4. Adya隔离级别体系

### 4.1 隔离级别定义

| Adya级别 | 禁止现象 | 定义 | 等价ANSI |
|----------|----------|------|----------|
| **PL-1** | G0 | 无脏写 | READ UNCOMMITTED |
| **PL-2** | G0, G1 | 无脏读 | READ COMMITTED |
| **PL-2.99** | G0, G1, G-SI | 快照隔离 | - |
| **PL-3** | G0, G1, G2 | 可串行化 | SERIALIZABLE |

### 4.2 形式化定义

**PL-1**:

$$
\text{PL-1} := \neg G0
$$

**PL-2**:

$$
\text{PL-2} := \neg G0 \land \neg G1
$$

**PL-2.99** (快照隔离):

$$
\text{PL-2.99} := \neg G0 \land \neg G1 \land \neg G\text{-}SI
$$

**PL-3**:

$$
\text{PL-3} := \neg G0 \land \neg G1 \land \neg G2
$$

### 4.3 隔离级别性质

**定理 4.1 (隔离级别层次)**:

$$
\text{PL-3} \subset \text{PL-2.99} \subset \text{PL-2} \subset \text{PL-1}
$$

**定理 4.2 (PL-3等价性)**:

$$
\text{PL-3} \equiv \text{Conflict Serializable}
$$

### 4.4 写倾斜 (Write Skew)

写倾斜是快照隔离下的典型异常：

```
T1: r(x=50) --------> w(y=60) ----> commit

         snapshot(x=50,y=50)

T2: r(y=50) --------> w(x=60) ----> commit

约束: x + y = 100

结果: x=60, y=60，违反约束!
```

DSG分析:

```
T1 →rw(y)→ T2 →rw(x)→ T1  (G2现象!)
```

在PL-2.99下允许，在PL-3下禁止。

## 5. 与ANSI SQL隔离级别对比

### 5.1 ANSI SQL标准

ANSI SQL基于现象定义隔离级别：

| ANSI级别 | 禁止现象 | 问题 |
|----------|----------|------|
| READ UNCOMMITTED | 无 | 允许脏读 |
| READ COMMITTED | P1 (脏读) | 允许不可重复读 |
| REPEATABLE READ | P1, P2 | 允许幻读 |
| SERIALIZABLE | P1, P2, P3 | - |

其中:

- **P1 (脏读)**: $w_i(x) \prec r_j(x) \prec a_i$
- **P2 (不可重复读)**: $r_i(x) \prec w_j(x) \prec c_j \prec r_i'(x)$
- **P3 (幻读)**: $r_i(P) \prec w_j(y \in P) \prec c_j \prec r_i'(P)$

### 5.2 ANSI vs Adya对比

| 维度 | ANSI SQL | Adya模型 |
|------|----------|----------|
| 基础 | 现象观察 | 依赖图分析 |
| 定义精度 | 模糊 | 精确 |
| 多版本支持 | 弱 | 强 |
| 可证明性 | 差 | 好 |

### 5.3 映射关系

```
ANSI标准              Adya模型              PostgreSQL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
READ UNCOMMITTED  →  PL-1              →  READ COMMITTED*
READ COMMITTED    →  PL-2              →  READ COMMITTED
REPEATABLE READ   →  PL-2.99 (近似)     →  REPEATABLE READ
SERIALIZABLE      →  PL-3              →  SERIALIZABLE (SSI)

* PostgreSQL中READ UNCOMMITTED被映射为READ COMMITTED
```

### 5.4 实现差异

**PostgreSQL实现**:

| PG隔离级别 | 实际Adya级别 | 实现机制 |
|------------|--------------|----------|
| READ COMMITTED | PL-2 | MVCC + 语句级快照 |
| REPEATABLE READ | PL-2.99 | MVCC + 事务级快照 |
| SERIALIZABLE | PL-3 | SSI (检测rw依赖环) |

## 6. PostgreSQL SSI实现深度分析

### 6.1 SSI原理

可串行化快照隔离 (Serializable Snapshot Isolation) 结合了:

1. 快照隔离的读取性能
2. 可串行化的正确性保证

**核心思想**: 检测rw-反依赖环，而非阻止所有冲突。

### 6.2 实现架构

```c
// src/backend/storage/lmgr/predicate.c

/*
 * SSI实现核心数据结构
 */

typedef struct SERIALIZABLEXACT {
    TransactionId xid;              /* 事务ID */
    SerCommitSeqNo commitSeqNo;     /* 提交序列号 */
    uint32 flags;                   /* 状态标志 */

    /* rw-依赖追踪 */
    SHM_QUEUE outConflicts;         /* 作为reader的冲突 */
    SHM_QUEUE inConflicts;          /* 作为writer的冲突 */

    /* 事务大小估计 (用于牺牲选择) */
    uint64 txnSize;

    /* 快照信息 */
    Snapshot snapshot;
} SERIALIZABLEXACT;
```

### 6.3 冲突检测

**检测场景**:

1. **事务 $T_j$ 写入 $T_i$ 读取过的数据**:
   - 创建rw-反依赖: $T_i \xrightarrow{rw} T_j$

2. **检查是否形成环**:
   - 如果 $T_j$ 已经有到 $T_i$ 的路径，则形成环

```c
// src/backend/storage/lmgr/predicate.c

/*
 * 检查rw-依赖是否形成环
 */
bool CheckForSerializationFailure(SERIALIZABLEXACT *reader,
                                   SERIALIZABLEXACT *writer) {
    // 1. 记录rw依赖: reader 读取了 writer 将要覆盖的数据
    SSIInsertConflict(reader, writer);

    // 2. 检测是否形成环
    if (SSICheckConflictLoop(reader, writer)) {
        // 3. 如果形成环，标记为"注定失败"
        if (reader->txnSize < writer->txnSize) {
            reader->flags |= SXACT_FLAG_DOOMED;
        } else {
            writer->flags |= SXACT_FLAG_DOOMED;
        }
        return true;
    }
    return false;
}
```

### 6.5 安全与危险结构

**安全结构** (Safe Structures):

```
T1 →rw→ T2 →wr→ T3   (安全)
```

**危险结构** (Dangerous Structures):

```
T1 →rw→ T2 →wr→ T3
          ↑______│
          (rw)
```

形成环，需中止一个事务。

### 6.6 优化策略

**只读事务优化**:

```c

if (sxact->flags & SXACT_FLAG_READ_ONLY) {
    return;  // 跳过冲突检测
}
```

**安全快照**:

```c
// 如果事务看到的状态已经是可串行化的
// 可以跳过冲突检测
if (IsSafeSnapshot(snapshot)) {
    sxact->flags |= SXACT_FLAG_SAFE_SNAPSHOT;
}
```

## 7. 形式化证明

### 7.1 SSI正确性

**定理 7.1 (SSI正确性)**:

SSI只允许可串行化的执行。

**证明**:

1. SSI检测所有rw-反依赖: $\forall T_i \xrightarrow{rw} T_j$，都记录在DSG中
2. 如果DSG包含rw-边的环，中止其中一个事务
3. 剩余历史无G2现象: $\neg G2$
4. 根据Adya定理: $\neg G2 \Rightarrow PL\text{-}3$ (可串行化)
5. ∎

### 7.2 冲突检测完备性

**定理 7.2 (检测完备性)**:

SSI检测所有违反可串行化的执行。

**证明草图**:

1. 设 $H$ 为不可串行化的执行
2. 则 $DSG(H)$ 存在环 $C$
3. 根据环的性质，$C$ 至少包含一条rw-边 (否则为G0或G1，已被快照隔离阻止)
4. SSI追踪所有rw-依赖
5. 当形成环时，至少有一条rw-边完成闭环
6. 此时SSI检测并中止
7. ∎

### 7.3 性能保证

**定理 7.3 (假阳性率)**:

SSI可能中止可串行化的执行 (假阳性)。

**示例**:

```
T1: r(x) ----> w(y) ----> commit
       \         /
        \       /
         w(x) /
T2: r(y) ----> commit

DSG: T1 →rw(y)→ T2 →rw(x)→ T1  (环!)

但调度实际可串行化: T1 → T2
```

这就是SSI的"保守性"，以换取高效实现。

## 8. 实际案例与异常检测

### 8.1 写倾斜案例分析

**场景**: 医院值班系统

```sql
-- 约束: 至少一名医生值班
CREATE TABLE doctors (
    name TEXT PRIMARY KEY,
    on_call BOOLEAN NOT NULL
);

INSERT INTO doctors VALUES ('Alice', true), ('Bob', true);
```

**事务**:

```sql
-- T1: Alice检查有两人在值班，自己下夜班
BEGIN ISOLATION LEVEL REPEATABLE READ;
SELECT * FROM doctors WHERE on_call = true;  -- 看到2人
UPDATE doctors SET on_call = false WHERE name = 'Alice';
COMMIT;

-- T2: Bob同时检查，也下夜班
BEGIN ISOLATION LEVEL REPEATABLE READ;
SELECT * FROM doctors WHERE on_call = true;  -- 也看到2人
UPDATE doctors SET on_call = false WHERE name = 'Bob';
COMMIT;
```

**结果**: 两人都下夜班，违反约束!

**DSG分析**:

```
T1 --rw(doctors)--> T2 --rw(doctors)--> T1  (G2!)
```

**解决方案**:

```sql
-- 方案1: SERIALIZABLE
BEGIN ISOLATION LEVEL SERIALIZABLE;
-- 其中一个事务将被中止

-- 方案2: 显式锁
SELECT * FROM doctors WHERE on_call = true FOR UPDATE;

-- 方案3: 约束检查
UPDATE doctors SET on_call = false
WHERE name = 'Alice'
AND (SELECT count(*) FROM doctors WHERE on_call = true) >= 2;
```

### 8.2 性能监控

**监控SSI冲突**:

```sql
-- 查看SSI冲突统计
SELECT
    pg_stat_get_db_conflict_serializable(oid) as conflicts
FROM pg_database
WHERE datname = current_database();

-- 查看SSI死锁
SELECT * FROM pg_stat_database
WHERE conflicts_serializable > 0;
```

### 8.3 调优建议

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 高SSI冲突率 | 写倾斜频繁 | 使用显式锁或调整业务逻辑 |
| 只读事务被中止 | 读取了脏数据 | 使用REPEATABLE READ |
| 长事务冲突 | 持锁时间过长 | 缩短事务，批量处理 |
