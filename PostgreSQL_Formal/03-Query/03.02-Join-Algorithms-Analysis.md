## 1. 概念定义

### 1.1 连接操作

关系 $R$ 和 $S$ 在属性 $A$ 上的连接:

$$
R \bowtie_{A} S := \{(r, s) \mid r \in R \land s \in S \land r[A] = s[A]\}
$$

### 1.2 连接算法对比

| 算法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|----------|
| **Nested Loop** | $O(\|R\| \times \|S\|)$ | $O(1)$ | 小表，索引可用 |
| **Hash Join** | $O(\|R\| + \|S\|)$ | $O(\|R\|)$ | 大表，等值连接 |
| **Merge Join** | $O(\|R\| \log \|R\| + \|S\| \log \|S\|)$ | $O(1)$ | 已排序数据 |

## 2. 算法详解

### 2.1 Hash Join

**算法**:

```
HashJoin(R, S):
    # 构建阶段
    H = new HashTable
    for r in R:
        key = hash(r.join_key)
        H[key].append(r)

    # 探测阶段
    for s in S:
        key = hash(s.join_key)
        for r in H[key]:
            if r.join_key == s.join_key:
                output(r, s)
```

**复杂度**:

- 构建: $O(|R|)$
- 探测: $O(|S| + |output|)$
- 总: $O(|R| + |S|)$

### 2.2 Merge Join

**前提**: 输入已排序

**算法**:

```
MergeJoin(R, S):
    r = R.first()
    s = S.first()

    while r and s:
        if r.key < s.key:
            r = R.next()
        else if r.key > s.key:
            s = S.next()
        else:
            # 匹配
            output(r, s)
            r = R.next()
```

## 3. PostgreSQL实现

### 3.1 连接选择

PostgreSQL优化器基于代价选择连接算法:

```c
// src/backend/optimizer/path/joinpath.c

// 如果R小且S有索引 -> Nested Loop
// 如果R和S大且等值 -> Hash Join
// 如果R和S已排序 -> Merge Join
```

### 3.2 配置参数

```ini
enable_nestloop = on
enable_hashjoin = on
enable_mergejoin = on

# 工作内存影响Hash Join
work_mem = 256MB
```

## 4. 反例分析

### 4.1 常见误用

**反例 1: 大数据集NestedLoop**

```sql
-- 两张大表使用Nested Loop
SELECT * FROM big_table b JOIN huge_table h ON b.id = h.id;

-- 解决方案: 禁用nestloop让优化器选择Hash Join
SET enable_nestloop = off;
```

**反例 2: work_mem不足**

```sql
-- Hash Join需要内存超过work_mem
-- 导致磁盘溢出，性能急剧下降

-- 解决方案
SET work_mem = '1GB';
```
