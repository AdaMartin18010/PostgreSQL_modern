# PostgreSQL 升级路径指南

> 📅 最后更新: 2026-04-07
>
> 本指南提供 PostgreSQL 版本升级的推荐路径、兼容性信息和风险评估。

---

## 升级策略概览

### 支持的升级方式

PostgreSQL 支持以下升级方式：

| 方式 | 适用场景 | 停机时间 | 复杂度 |
|------|----------|----------|--------|
| **pg_upgrade** | 同机升级，版本跨度小 | 分钟级 | 低 |
| **逻辑复制** | 跨版本，大版本跳跃 | 秒级 | 中 |
| **pg_dump/pg_restore** | 跨版本，数据迁移 | 小时级 | 低 |
| **pg_dumpall** | 整个集群迁移 | 小时级 | 低 |

## 推荐的升级路径

### 标准升级路径 (推荐)

```
PG9.5/9.6 ──────────────────────────────────────────────────►
    │                                                          │
    ▼                                                          ▼
  PG10 ──► PG11 ──► PG12 ──► PG13 ──► PG14 ──► PG15 ──► PG16 ──► PG17+
    │       │        │        │        │        │        │
    │       │        │        │        │        │        └── SQL/JSON, 逻辑复制
    │       │        │        │        │        └── MERGE, 压缩
    │       │        │        │        └── Multirange, 性能
    │       │        │        └── 增量排序, 分区复制
    │       │        └── JIT编译, 生成列
    │       └── 哈希分区, 存储过程
    └── 声明式分区, 逻辑复制
```

### 快速升级路径 (适用于稳定环境)

```
PG10/11 ───────► PG14 ───────► PG17+
     │              │             │
     └─ 中间版本    └─ 验证点     └─ 最新稳定版

风险: 跳过中间版本可能遗漏重要的行为变更警告
```

### 保守升级路径 (适用于关键业务)

```
PG10 ──► PG12 ──► PG14 ──► PG16
  │        │        │        │
  │        │        │        └── 长期支持版本
  │        │        └── 广泛验证版本
  │        └── JIT稳定版本
  └── 起始版本
```

## 版本跳跃升级的兼容性

### 直接升级兼容性矩阵

| 从版本 | 可直接升级到 | 推荐方式 | 风险等级 |
|--------|--------------|----------|----------|
| **PG9.5** | PG10-PG13 | pg_dump/pg_restore | 🟡 中 |
| **PG9.6** | PG10-PG13 | pg_dump/pg_restore | 🟡 中 |
| **PG10** | PG11-PG17+ | pg_upgrade / 逻辑复制 | 🟢 低 |
| **PG11** | PG12-PG17+ | pg_upgrade / 逻辑复制 | 🟢 低 |
| **PG12** | PG13-PG17+ | pg_upgrade / 逻辑复制 | 🟢 低 |
| **PG13** | PG14-PG17+ | pg_upgrade / 逻辑复制 | 🟢 低 |
| **PG14** | PG15-PG17+ | pg_upgrade / 逻辑复制 | 🟢 低 |
| **PG15** | PG16-PG17+ | pg_upgrade / 逻辑复制 | 🟢 低 |
| **PG16** | PG17+ | pg_upgrade / 逻辑复制 | 🟢 低 |

> **注意**: PostgreSQL 官方支持跨版本升级，但建议阅读所有中间版本的发布说明。

## 各版本跳跃升级注意事项

### PG10 之前版本升级

| 起始版本 | 关键注意事项 |
|----------|--------------|
| PG9.5 | `pg_authid` 系统表变更，需重建视图 |
| PG9.6 | 并行查询参数默认值变更，需测试查询计划 |

### PG10 → PG11

| 检查项 | 详情 |
|--------|------|
| 哈希分区 | 新增功能，可优化大表分区策略 |
| 存储过程 | 新增，可替换部分函数 |
| 默认分区 | 分区表建议添加默认分区 |

### PG11 → PG12

| 检查项 | 详情 |
|--------|------|
| JIT 编译 | 默认启用，需验证性能影响 |
| CTE 优化 | CTE 默认可内联，可能改变查询行为 |
| 生成列 | 新特性，可考虑替代触发器方案 |

### PG12 → PG13

| 检查项 | 详情 |
|--------|------|
| 增量排序 | 可能改变查询计划，需测试 |
| 复制槽 | 检查逻辑复制槽兼容性 |

### PG13 → PG14

| 检查项 | 详情 |
|--------|------|
| Multirange | 新类型，应用可评估使用 |
| 查询优化器 | 统计信息收集变更 |

### PG14 → PG15

| 检查项 | 详情 |
|--------|------|
| MERGE 命令 | 可利用替换 ON CONFLICT |
| 排序内存 | 排序算法变更，work_mem 可能需要调整 |
| 权限变更 | `CREATEROLE` 权限更严格 |
| 压缩选项 | WAL 和备份支持 LZ4/zstd |

### PG15 → PG16

| 检查项 | 详情 |
|--------|------|
| SQL/JSON | 新构造器函数可用 |
| 逻辑复制 | 可从备库复制，架构更灵活 |
| VACUUM | 并行 VACUUM 增强 |
| 权限角色 | 新增 `pg_create_subscription` |

### PG16 → PG17

| 检查项 | 详情 |
|--------|------|
| VACUUM 内存 | 内存使用优化 |
| 增量备份 | 支持增量备份，需评估策略 |
| JSON_TABLE | SQL/JSON 标准支持 |
| MERGE RETURNING | MERGE 支持 RETURNING 子句 |

## 风险等级评估

### 🟢 低风险 (建议升级)

| 条件 | 说明 |
|------|------|
| 版本跨度 ≤ 3 | 跳过的版本较少 |
| 有完整测试环境 | 可在测试环境充分验证 |
| 使用 pg_upgrade | 停机时间短，回滚简单 |
| 无自定义扩展 | 减少兼容性问题 |

### 🟡 中风险 (谨慎升级)

| 条件 | 说明 |
|------|------|
| 版本跨度 4-6 | 需要阅读大量发布说明 |
| 使用逻辑复制 | 需要额外配置和监控 |
| 有自定义扩展 | 需验证扩展兼容性 |
| 生产环境 | 需要制定详细的回滚方案 |

### 🔴 高风险 (需要详细规划)

| 条件 | 说明 |
|------|------|
| 版本跨度 > 6 | 可能遗漏重要变更 |
| PG9.x 版本 | 系统表结构差异大 |
| 复杂分区表 | 分区机制多次变更 |
| 关键业务系统 | 停机成本极高 |

## 升级前检查清单

### 系统检查

- [ ] 确认当前版本和目标版本的兼容性
- [ ] 检查磁盘空间 (至少预留 1.5 倍数据空间)
- [ ] 验证操作系统和库依赖
- [ ] 确认扩展兼容性

### 数据检查

- [ ] 运行 `pg_dumpall --schema-only` 备份结构
- [ ] 检查无效索引 `SELECT * FROM pg_class WHERE relkind = 'i' AND NOT relisvalid`
- [ ] 检查孤立对象
- [ ] 验证字符集兼容性

### 应用检查

- [ ] 检查 SQL 语法兼容性
- [ ] 验证连接字符串
- [ ] 测试连接池配置
- [ ] 确认驱动版本兼容性

### 配置检查

- [ ] 对比配置参数变更
- [ ] 检查新增/删除的参数
- [ ] 验证内存设置合理性

## 升级后优化建议

### 立即执行

```sql
-- 更新统计信息
VACUUM ANALYZE;

-- 验证数据一致性
SELECT pg_catalog.pg_check_integrity();
```

### 可选优化

```sql
-- 重建索引 (特别是从旧版本升级)
REINDEX DATABASE your_database;

-- 更新扩展
ALTER EXTENSION postgis UPDATE;
```
