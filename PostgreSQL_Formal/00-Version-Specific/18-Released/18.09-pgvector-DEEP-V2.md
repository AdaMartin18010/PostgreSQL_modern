## 1. 向量数据库理论基础

### 1.1 向量空间模型

向量数据库的核心是将非结构化数据（文本、图像、音频等）映射到高维向量空间中的点。
这种映射通过深度学习模型（如BERT、GPT、ResNet）实现，将语义信息编码为固定维度的稠密向量。

#### 1.1.1 向量表示原理

给定输入数据 $x$，嵌入模型 $f_\theta$ 将其转换为 $d$ 维向量：

$$\mathbf{v} = f_\theta(x) \in \mathbb{R}^d$$

其中：

- $\mathbf{v}$ 是输出向量
- $d$ 是向量维度（常见值：128, 256, 384, 512, 768, 1024, 1536, 2048）
- $\theta$ 是模型参数

**维度与表达能力的关系：**

信息容量上限由维度决定：

$$C_{max} = \log_2(2^d) = d \text{ bits}$$

实际有效信息受模型能力限制：

$$C_{effective} = d \cdot H(\mathbf{v})$$

其中 $H(\mathbf{v})$ 是向量的平均信息熵：

$$H(\mathbf{v}) = -\frac{1}{d}\sum_{i=1}^{d} p(v_i) \log_2 p(v_i)$$

#### 1.1.2 向量空间的数学性质

向量空间 $(\mathbb{R}^d, +, \cdot)$ 满足以下公理：

**加法封闭性：**
$$\forall \mathbf{u}, \mathbf{v} \in \mathbb{R}^d: \mathbf{u} + \mathbf{v} \in \mathbb{R}^d$$

**标量乘法：**
$$\forall \alpha \in \mathbb{R}, \mathbf{v} \in \mathbb{R}^d: \alpha \cdot \mathbf{v} \in \mathbb{R}^d$$

**内积空间：**
向量空间配备内积运算 $\langle \cdot, \cdot \rangle: \mathbb{R}^d \times \mathbb{R}^d \rightarrow \mathbb{R}$：

$$\langle \mathbf{u}, \mathbf{v} \rangle = \sum_{i=1}^{d} u_i v_i$$

**柯西-施瓦茨不等式：**

$$|\langle \mathbf{u}, \mathbf{v} \rangle| \leq \|\mathbf{u}\| \cdot \|\mathbf{v}\|$$

当且仅当 $\mathbf{u}$ 和 $\mathbf{v}$ 线性相关时取等号。

#### 1.1.3 嵌入空间的流形假设

高维数据通常位于低维流形上。设数据流形 $\mathcal{M} \subset \mathbb{R}^d$，其本征维度 $d_{eff} \ll d$：

$$dim(\mathcal{M}) = d_{eff} = \inf\{k : \exists \phi: \mathcal{M} \rightarrow \mathbb{R}^k, \text{局部同胚}\}$$

局部线性嵌入（LLE）假设：

$$\mathbf{x}_i \approx \sum_{j \in \mathcal{N}(i)} w_{ij} \mathbf{x}_j$$

其中 $\mathcal{N}(i)$ 是 $k$ 近邻集合，$w_{ij}$ 是重构权重。

### 1.2 相似度度量

相似度度量是向量搜索的核心，定义了两个向量在语义空间中的接近程度。

#### 1.2.1 欧几里得距离 (L2 Distance)

欧氏距离测量向量空间中的直线距离：

$$d_{L2}(\mathbf{u}, \mathbf{v}) = \|\mathbf{u} - \mathbf{v}\|_2 = \sqrt{\sum_{i=1}^{d}(u_i - v_i)^2}$$

**平方欧氏距离**（计算优化）：

$$d_{L2}^2(\mathbf{u}, \mathbf{v}) = \sum_{i=1}^{d}(u_i - v_i)^2 = \|\mathbf{u}\|_2^2 + \|\mathbf{v}\|_2^2 - 2\langle \mathbf{u}, \mathbf{v} \rangle$$

当向量已归一化时，简化为：

$$d_{L2}^2(\mathbf{u}, \mathbf{v}) = 2 - 2\langle \mathbf{u}, \mathbf{v} \rangle$$

**性质：**

- 满足度量空间公理（非负性、对称性、三角不等式）
- 对绝对位置敏感
- 适用于特征绝对值有意义的场景

#### 1.2.2 余弦相似度 (Cosine Similarity)

余弦相似度测量向量方向的夹角余弦值：

$$\text{cos}(\mathbf{u}, \mathbf{v}) = \frac{\langle \mathbf{u}, \mathbf{v} \rangle}{\|\mathbf{u}\|_2 \cdot \|\mathbf{v}\|_2} = \frac{\sum_{i=1}^{d}u_i v_i}{\sqrt{\sum_{i=1}^{d}u_i^2} \cdot \sqrt{\sum_{i=1}^{d}v_i^2}}$$

**余弦距离：**

$$d_{cos}(\mathbf{u}, \mathbf{v}) = 1 - \text{cos}(\mathbf{u}, \mathbf{v})$$

**角度解释：**

$$\theta = \arccos(\text{cos}(\mathbf{u}, \mathbf{v}))$$

**与欧氏距离的关系（单位向量）：**

$$d_{L2}^2(\mathbf{u}, \mathbf{v}) = 2 - 2\text{cos}(\mathbf{u}, \mathbf{v}) = 2 \cdot d_{cos}(\mathbf{u}, \mathbf{v})$$

**性质：**

- 忽略向量模长，仅关注方向
- 范围 $[-1, 1]$，常用 $[0, 1]$（非负嵌入）
- 适用于语义相似度比较

#### 1.2.3 内积相似度 (Dot Product / Inner Product)

内积直接计算向量的点积：

$$\text{dot}(\mathbf{u}, \mathbf{v}) = \langle \mathbf{u}, \mathbf{v} \rangle = \sum_{i=1}^{d}u_i v_i$$

**与余弦相似度的关系：**

$$\text{dot}(\mathbf{u}, \mathbf{v}) = \|\mathbf{u}\|_2 \cdot \|\mathbf{v}\|_2 \cdot \text{cos}(\mathbf{u}, \mathbf{v})$$

**最大内积搜索（MIPS）：**

$$\mathbf{v}^* = \arg\max_{\mathbf{v} \in \mathcal{D}} \langle \mathbf{q}, \mathbf{v} \rangle$$

**应用场景：**

- 推荐系统（考虑偏好的强度）
- 需要考虑向量模长的相似度

#### 1.2.4 汉明距离 (Hamming Distance)

用于二进制向量：

$$d_H(\mathbf{u}, \mathbf{v}) = \sum_{i=1}^{d} \mathbb{I}[u_i \neq v_i]$$

**Jaccard相似度：**

$$J(\mathbf{u}, \mathbf{v}) = \frac{|\mathbf{u} \cap \mathbf{v}|}{|\mathbf{u} \cup \mathbf{v}|}$$

#### 1.2.5 相似度度量选择指南

| 度量方式 | 公式 | 适用场景 | 计算复杂度 |
|---------|------|---------|-----------|
| 欧氏距离 | $\|\mathbf{u}-\mathbf{v}\|_2$ | 绝对位置重要 | $O(d)$ |
| 余弦相似度 | $\frac{\langle\mathbf{u},\mathbf{v}\rangle}{\|\mathbf{u}\|\|\mathbf{v}\|}$ | 语义相似度 | $O(d)$ |
| 内积 | $\langle\mathbf{u},\mathbf{v}\rangle$ | 推荐系统 | $O(d)$ |
| 曼哈顿距离 | $\|\mathbf{u}-\mathbf{v}\|_1$ | 稀疏向量 | $O(d)$ |

### 1.3 ANN近似最近邻算法

精确最近邻（Exact NN）的时间复杂度为 $O(nd)$，对于大规模数据集不可行。近似最近邻（ANN）算法通过牺牲少量精度换取数量级性能提升。

#### 1.3.1 ANN问题定义

给定查询向量 $\mathbf{q}$ 和数据集 $\mathcal{D} = \{\mathbf{v}_1, ..., \mathbf{v}_n\}$，寻找近似最近邻：

$$\mathbf{v}_{ANN} \in \mathcal{D}: d(\mathbf{q}, \mathbf{v}_{ANN}) \leq c \cdot d(\mathbf{q}, \mathbf{v}_{NN})$$

其中 $c > 1$ 是近似因子，$\mathbf{v}_{NN}$ 是真实最近邻。

$(c, \delta)$-近似最近邻：

$$P\left[d(\mathbf{q}, \mathbf{v}_{ANN}) \leq c \cdot d(\mathbf{q}, \mathbf{v}_{NN})\right] \geq 1 - \delta$$

#### 1.3.2 局部敏感哈希（LSH）

LSH的核心思想：相似项以高概率碰撞，不相似项以低概率碰撞。

**定义：** 函数族 $\mathcal{H}$ 是 $(r, cr, p_1, p_2)$-敏感的，如果：

- 若 $d(\mathbf{u}, \mathbf{v}) \leq r$，则 $P_{h \in \mathcal{H}}[h(\mathbf{u}) = h(\mathbf{v})] \geq p_1$
- 若 $d(\mathbf{u}, \mathbf{v}) \geq cr$，则 $P_{h \in \mathcal{H}}[h(\mathbf{u}) = h(\mathbf{v})] \leq p_2$

**随机投影LSH：**

$$h_{\mathbf{a},b}(\mathbf{v}) = \left\lfloor \frac{\langle \mathbf{a}, \mathbf{v} \rangle + b}{w} \right\rfloor$$

其中 $\mathbf{a} \sim \mathcal{N}(0, \mathbf{I})$，$b \sim U(0, w)$。

**多表哈希：**

$$g(\mathbf{v}) = (h_1(\mathbf{v}), h_2(\mathbf{v}), ..., h_k(\mathbf{v}))$$

使用 $L$ 个独立哈希表，查询时检索所有候选。

#### 1.3.3 空间划分树

**KD-Tree：**

递归划分空间：

$$\text{split}_i = \text{median}(\{v_i : \mathbf{v} \in \mathcal{D}\})$$

查询复杂度（平衡树）：

$$T(n) = O(\log n + k)$$

高维诅咒：$d > 20$ 时效率下降为线性扫描。

**球树（Ball Tree）：**

使用超球体而非超矩形划分：

$$B(c, r) = \{\mathbf{v} : \|\mathbf{v} - c\| \leq r\}$$

利用三角不等式剪枝：

$$\text{若 } |\|\mathbf{q} - c\| - r| > d_{current}^{best}, \text{ 剪枝}$$

#### 1.3.4 图索引算法

**导航小世界（NSW）图：**

构建增量式近似Delaunay图：

$$G = (V, E), \quad |E| \approx M \cdot |V|$$

贪心搜索算法：

```
function GREEDY_SEARCH(q, enter_point):
    visited = {enter_point}
    candidates = {enter_point}
    while True:
        v = argmin_{u in candidates} d(q, u)
        if d(q, v) >= d(q, current_best):
            break
        for w in neighbors(v):
            if w not in visited:
                visited.add(w)
                candidates.add(w)
    return current_best
```

**分层可导航小世界（HNSW）：**

多层图结构：

$$G = \{G_0, G_1, ..., G_{L_{max}}\}$$

其中 $G_0$ 包含所有节点，上层稀疏采样：

$$P(level = l) = e^{-l/\lambda} \cdot (1 - e^{-1/\lambda})$$

期望层数：

$$E[L] = \frac{1}{1 - e^{-1/\lambda}} \approx \lambda$$

搜索复杂度：

$$T_{search} = O(\log n \cdot M_{ef})$$

#### 1.3.5 倒排索引（IVF）

**K-Means聚类：**

将空间划分为 $k$ 个Voronoi单元：

$$\mathcal{C}_i = \{\mathbf{v} : \|\mathbf{v} - \mu_i\| \leq \|\mathbf{v} - \mu_j\|, \forall j \neq i\}$$

目标函数：

$$J = \sum_{i=1}^{k} \sum_{\mathbf{v} \in \mathcal{C}_i} \|\mathbf{v} - \mu_i\|^2$$

**搜索过程：**

```
function IVF_SEARCH(q, nprobe):
    # 找到最近的nprobe个质心
    centroids = argmin_{i}^{nprobe} ||q - mu_i||
    candidates = union_{i in centroids} C_i
    # 在候选集中精确搜索
    return EXACT_SEARCH(q, candidates, k)
```

复杂度：

$$T_{search} = O(kd + \frac{n}{k} \cdot nprobe \cdot d)$$

最优 $k = \sqrt{n}$，此时 $T_{search} = O(\sqrt{n} \cdot d)$。

## 2. pgvector实现机制

### 2.1 扩展架构

pgvector是PostgreSQL的开源向量扩展，将向量作为原生数据类型支持。

#### 2.1.1 类型系统

```sql
-- 向量类型定义
CREATE TYPE vector AS (dim integer, data real[]);

-- 内部存储格式
/* 向量存储结构（MAXDIM = 16000）
 * Header: 4 bytes (dimensions)
 * Data: dim * 4 bytes (float32 values)
 */
```

向量维度限制：

$$dim_{max} = 16000$$

存储大小计算：

$$\text{size}(\mathbf{v}) = 4 + 4 \cdot dim(\mathbf{v}) \text{ bytes}$$

#### 2.1.2 运算符支持

```sql
-- 算术运算
SELECT '[1,2,3]'::vector + '[4,5,6]'::vector;  -- [5,7,9]
SELECT '[1,2,3]'::vector - '[1,1,1]'::vector;  -- [0,1,2]

-- 标量乘
SELECT 2.0 * '[1,2,3]'::vector;  -- [2,4,6]

-- 向量范数
SELECT l2_norm('[3,4]'::vector);  -- 5.0
```

### 2.2 IVFFlat索引

IVFFlat（倒排文件扁平索引）是pgvector的默认索引类型，基于K-Means聚类。

#### 2.2.1 索引构建

```sql
-- 创建IVFFlat索引
CREATE INDEX ON items
USING ivfflat (embedding vector_cosine_ops)
WITH (lists = 100);
```

**参数选择公式：**

最优列表数与数据量关系：

$$lists = \sqrt{n}$$

其中 $n$ 是向量数量。

**内存占用估算：**

$$\text{Memory}_{IVFFlat} = k \cdot d \cdot 4 + n \cdot d \cdot 4 \text{ bytes}$$

聚类质心存储 + 向量存储（TOAST压缩后可能更小）。

#### 2.2.2 聚类算法

**K-Means++初始化：**

```sql
-- 质心选择概率
P(c_i) = \frac{D(c_i)^2}{\sum_{j} D(c_j)^2}
```

**Lloyd迭代：**

$$\mu_i^{(t+1)} = \frac{1}{|\mathcal{C}_i^{(t)}|} \sum_{\mathbf{v} \in \mathcal{C}_i^{(t)}} \mathbf{v}$$

收敛条件：

$$|J^{(t)} - J^{(t+1)}| < \epsilon$$

#### 2.2.3 查询优化

```sql
-- 设置探测列表数（默认1，建议根据精度需求调整）
SET ivfflat.probes = 10;

-- 查询示例
SELECT id, embedding <=> query_embedding AS distance
FROM items
ORDER BY embedding <=> query_embedding
LIMIT 10;
```

**探测列表数与召回率关系：**

$$Recall \approx 1 - e^{-\alpha \cdot nprobe}$$

其中 $\alpha$ 是与数据分布相关的常数。

**查询时间复杂度：**

$$T_{query} = O(kd + \frac{n \cdot nprobe}{k} \cdot d)$$

#### 2.2.4 距离运算符

| 运算符 | 含义 | 支持的距离类型 |
|-------|------|--------------|
| `<->` | L2距离 | 欧氏距离 |
| `<#>` | 负内积 | 内积（最大内积搜索） |
| `<=>` | 余弦距离 | 1 - 余弦相似度 |

```sql
-- L2距离查询
SELECT * FROM items
ORDER BY embedding <-> '[1,2,3]'::vector
LIMIT 5;

-- 余弦相似度查询（ORDER BY余弦距离）
SELECT * FROM items
ORDER BY embedding <=> '[1,2,3]'::vector
LIMIT 5;

-- 内积查询（负值，需反转ORDER方向）
SELECT * FROM items
ORDER BY embedding <#> '[1,2,3]'::vector
LIMIT 5;
```

### 2.3 HNSW索引

HNSW（分层可导航小世界）是pgvector 0.5.0+支持的高性能索引，提供更高的搜索精度。

#### 2.3.1 索引构建

```sql
-- 创建HNSW索引
CREATE INDEX ON items
USING hnsw (embedding vector_cosine_ops)
WITH (m = 16, ef_construction = 64);
```

**参数说明：**

- `m`: 每层最大连接数（默认16，推荐8-32）
- `ef_construction`: 构建时搜索队列大小（默认64，推荐64-128）

**参数与性能关系：**

索引大小：

$$\text{Size}_{HNSW} \approx n \cdot m \cdot (4 + 4 + 4) \text{ bytes} = 12 \cdot n \cdot m \text{ bytes}$$

构建时间：

$$T_{build} = O(n \cdot \log n \cdot ef\_construction)$$

#### 2.3.2 图构建算法

```sql
-- 伪代码：HNSW插入算法
FUNCTION INSERT(q, m, M_max, efConstruction):
    W = SEARCH_LAYER(q, enter_point, ef=efConstruction, l=layer(q))
    neighbors = SELECT_NEIGHBORS(q, W, m, layer=0)
    ADD_EDGES(q, neighbors)

    FOR each e IN neighbors:
        IF degree(e) > M_max:
            new_neighbors = SELECT_NEIGHBORS(e, neighbors(e), M_max)
            UPDATE_EDGES(e, new_neighbors)
```

**邻居选择启发式（Heuristic）：**

```sql
-- 启发式选择考虑距离和连通性
FUNCTION SELECT_NEIGHBORS_HEURISTIC(q, C, M):
    R = empty_set
    W = C  // 候选集

    WHILE |W| > 0 AND |R| < M:
        e = argmin_{e in W} ||q - e||
        IF R is empty OR ||e - q|| < min_{r in R} ||e - r||:
            R = R UNION {e}
        W = W - {e}

    RETURN R
```

#### 2.3.3 分层搜索

```sql
-- 搜索参数
SET hnsw.ef_search = 100;

-- 查询
SELECT id, embedding <=> query_vec AS dist
FROM items
ORDER BY embedding <=> query_vec
LIMIT k;
```

**搜索复杂度：**

$$T_{search} = O(\log n \cdot ef\_search)$$

**ef_search与召回率：**

$$Recall@k \geq 1 - e^{-\beta \cdot (ef\_search - k)}$$

推荐设置：

$$ef\_search \geq k + \Delta, \quad \Delta \in [10, 50]$$

| 特性 | IVFFlat | HNSW |

### 2.4 向量运算优化

#### 2.4.1 SIMD加速

pgvector支持SIMD指令集加速向量运算：

```c
// AVX-512 加速（512位向量寄存器）
// 单次操作可处理 512/32 = 16 个float32

#ifdef __AVX512F__
    __m512 sum = _mm512_setzero_ps();
    for (int i = 0; i < dim; i += 16) {
        __m512 a = _mm512_loadu_ps(&u[i]);
        __m512 b = _mm512_loadu_ps(&v[i]);
        sum = _mm512_fmadd_ps(a, b, sum);  // FMA: a*b + sum
    }
    dot = _mm512_reduce_add_ps(sum);
#endif
```

加速比：

$$\text{Speedup}_{AVX512} = \frac{d}{\lceil d/16 \rceil \cdot (1 + \frac{1}{16})} \approx 14-15\times$$

#### 2.4.2 量化优化

**乘积量化（Product Quantization）：**

将向量分割为 $m$ 个子空间：

$$\mathbf{v} = (\mathbf{v}^1, \mathbf{v}^2, ..., \mathbf{v}^m), \quad \mathbf{v}^j \in \mathbb{R}^{d/m}$$

每个子空间独立量化：

$$q(\mathbf{v}^j) = \arg\min_{c \in \mathcal{C}_j} \|\mathbf{v}^j - c\|$$

压缩率：

$$\text{Compression} = \frac{32 \cdot d}{m \cdot \log_2 k} = \frac{32d}{m \cdot b}$$

其中 $k = 2^b$ 是每个子空间的码本大小。

### 3.2 RAG检索增强生成

#### 3.2.1 RAG架构设计

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│   User      │────▶│   Query      │────▶│   Embedding │
│   Query     │     │   Processing │     │   Model     │
└─────────────┘     └──────────────┘     └─────────────┘
                                                   │
                                                   ▼
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│   LLM       │◀────│   Context    │◀────│   Vector    │
│   Response  │     │   Assembly   │     │   Search    │
└─────────────┘     └──────────────┘     └─────────────┘
```

**RAG概率模型：**

$$P_{RAG}(y|x) = \sum_{z \in \mathcal{R}(x)} P(y|x, z) \cdot P(z|x)$$

其中：

- $x$: 用户查询
- $z$: 检索到的文档
- $y$: 生成的回复
- $\mathcal{R}(x)$: 检索结果集合

#### 3.2.2 检索策略

```python
import openai
import psycopg2

class RAGEngine:
    def __init__(self, db_conn, embedding_model="text-embedding-3-large"):
        self.conn = db_conn
        self.embedding_model = embedding_model

    def generate_query_embedding(self, query):
        """生成查询嵌入"""
        response = openai.Embedding.create(
            input=query,
            model=self.embedding_model
        )
        return response['data'][0]['embedding']

    def hybrid_search(self, query, embedding, top_k=5, keyword_weight=0.3):
        """
        混合搜索：结合向量相似度和关键词匹配

        分数计算：
        score = (1 - α) * cosine_score + α * ts_rank
        """
        cur = self.conn.cursor()

        cur.execute("""
            WITH vector_scores AS (
                SELECT
                    id,
                    content,
                    1 - (embedding <=> %s::vector) AS vec_score
                FROM documents
                ORDER BY embedding <=> %s::vector
                LIMIT %s * 2
            ),
            keyword_scores AS (
                SELECT
                    id,
                    ts_rank_cd(
                        to_tsvector('english', content),
                        plainto_tsquery('english', %s),
                        32
                    ) AS kw_score
                FROM documents
                WHERE to_tsvector('english', content) @@
                      plainto_tsquery('english', %s)
            )
            SELECT
                v.id,
                v.content,
                (1 - %s) * v.vec_score + %s * COALESCE(k.kw_score, 0) AS score
            FROM vector_scores v
            LEFT JOIN keyword_scores k ON v.id = k.id
            ORDER BY score DESC
            LIMIT %s
        """, (embedding, embedding, top_k, query, query,
              keyword_weight, keyword_weight, top_k))

        return cur.fetchall()

    def mmr_search(self, query_embedding, top_k=5, lambda_param=0.5):
        """
        MMR (Maximal Marginal Relevance) 搜索
        平衡相关性和多样性

        MMR = λ * Sim(q, d) - (1-λ) * max Sim(d, d_selected)
        """
        cur = self.conn.cursor()

        # 获取候选集
        cur.execute("""
            SELECT id, content, embedding,
                   1 - (embedding <=> %s::vector) AS relevance
            FROM documents
            ORDER BY embedding <=> %s::vector
            LIMIT 50
        """, (query_embedding, query_embedding))

        candidates = cur.fetchall()

        # MMR选择
        selected = []
        selected_embeddings = []

        while len(selected) < top_k and candidates:
            best_mmr = -float('inf')
            best_idx = -1

            for i, (id, content, emb, rel) in enumerate(candidates):
                if selected_embeddings:
                    max_sim = max(
                        self.cosine_sim(emb, s_emb)
                        for s_emb in selected_embeddings
                    )
                else:
                    max_sim = 0

                mmr_score = lambda_param * rel - (1 - lambda_param) * max_sim

                if mmr_score > best_mmr:
                    best_mmr = mmr_score
                    best_idx = i

            selected.append(candidates[best_idx])
            selected_embeddings.append(candidates[best_idx][2])
            candidates.pop(best_idx)

        return selected

    @staticmethod
    def cosine_sim(a, b):
        """计算余弦相似度"""
        return sum(x*y for x, y in zip(a, b)) / (
            sum(x*x for x in a) ** 0.5 * sum(x*x for x in b) ** 0.5
        )
```

```python
def assemble_context(retrieved_docs, max_tokens=4000, tokenizer=None):
    """
    组装检索文档为上下文
    使用贪心算法最大化信息密度
    """
    if tokenizer is None:
        tokenizer = tiktoken.get_encoding("cl100k_base")

    context_parts = []
    current_tokens = 0

    for doc_id, content, score in retrieved_docs:
        doc_tokens = len(tokenizer.encode(content))

        if current_tokens + doc_tokens > max_tokens:
            break

        context_parts.append({
            'id': doc_id,
            'content': content,
            'score': score,
            'tokens': doc_tokens
        })
        current_tokens += doc_tokens

    # 格式化上下文
    formatted_context = "\n\n---\n\n".join([
        f"[Document {i+1}] (Relevance: {p['score']:.3f})\n{p['content']}"
        for i, p in enumerate(context_parts)
    ])

    return formatted_context, context_parts

def generate_rag_response(query, context, model="gpt-4"):

{context}

Answer:"""

    response = openai.ChatCompletion.create(

        messages=[
            {"role": "system", "content": "You are a helpful assistant."},
            {"role": "user", "content": prompt}
        ],
        temperature=0.3,
        max_tokens=1000
    )
```

```sql
    p_final_k INT DEFAULT 5
```

```
         │        └── 定期重建索引
         └── 否 → 需要最高查询性能?
                  ├── 是 → HNSW
                  │        ├── m = 16-32
                  │        └── ef_construction = 64-128
                  └── 否 → IVFFlat (平衡方案)
```

#### 4.1.2 参数选择公式

**IVFFlat参数：**

```
lists = √n （n为向量数量）

示例：
- n = 100,000 → lists = 316 → 取整为 300
- n = 1,000,000 → lists = 1000
- n = 10,000,000 → lists = 3162 → 考虑分区
```

**HNSW参数：**

```
m = max(8, min(64, log10(n)))
ef_construction = max(64, min(512, 2 * m))
ef_search = max(k + 10, 50)  # k为查询返回数量

示例：
- n = 100,000 → m=16, ef_construction=64
- n = 1,000,000 → m=20, ef_construction=100
- n = 10,000,000 → m=24, ef_construction=128
```

```sql
-- 动态索引创建（根据数据量自动选择参数）
CREATE OR REPLACE FUNCTION create_optimal_vector_index(
    p_table_name TEXT,
    p_column_name TEXT,
    p_metric TEXT DEFAULT 'cosine'  -- 'cosine', 'l2', 'ip'
)
RETURNS TEXT AS $$
DECLARE
    v_count BIGINT;
    v_lists INT;
    v_m INT;
    v_ef_construction INT;
    v_ops TEXT;
    v_sql TEXT;
BEGIN
    -- 获取向量数量
    EXECUTE format('SELECT COUNT(*) FROM %I', p_table_name) INTO v_count;

    -- 选择运算符类
    v_ops := CASE p_metric
        WHEN 'cosine' THEN 'vector_cosine_ops'
        WHEN 'l2' THEN 'vector_l2_ops'
        WHEN 'ip' THEN 'vector_ip_ops'
        ELSE 'vector_cosine_ops'
    END;

    -- 根据数据量选择索引类型和参数
    IF v_count < 100000 THEN
        -- 使用IVFFlat
        v_lists := GREATEST(10, ROUND(SQRT(v_count))::INT);
        v_sql := format(
            'CREATE INDEX ON %I USING ivfflat (%I %s) WITH (lists = %s)',
            p_table_name, p_column_name, v_ops, v_lists
        );
        RAISE NOTICE 'Using IVFFlat with lists = %', v_lists;
    ELSE
        -- 使用HNSW
        v_m := GREATEST(8, LEAST(64, ROUND(LOG(10, v_count))::INT));
        v_ef_construction := GREATEST(64, 2 * v_m);
        v_sql := format(
            'CREATE INDEX ON %I USING hnsw (%I %s) WITH (m = %s, ef_construction = %s)',
            p_table_name, p_column_name, v_ops, v_m, v_ef_construction
        );
        RAISE NOTICE 'Using HNSW with m = %, ef_construction = %', v_m, v_ef_construction;
    END IF;

    EXECUTE v_sql;

    RETURN v_sql;
END;
$$ LANGUAGE plpgsql;

-- 使用示例
```

### 4.2 维度优化

#### 4.2.1 降维技术

```python
import numpy as np
from sklearn.decomposition import PCA
from sklearn.random_projection import SparseRandomProjection
import psycopg2

def optimize_embedding_dimension(
    conn,
    table_name,
    column_name,
    target_variance=0.95,
    max_samples=100000
):
    """
    使用PCA确定最优维度
    """
    cur = conn.cursor()

    # 采样数据
    cur.execute(f"""
        SELECT {column_name}
        FROM {table_name}
        TABLESAMPLE SYSTEM (10)
        LIMIT %s
    """, (max_samples,))

    embeddings = np.array([row[0] for row in cur.fetchall()])

    # PCA分析
    pca = PCA()
    pca.fit(embeddings)

    # 找到保留目标方差的最小维度
    cumsum_variance = np.cumsum(pca.explained_variance_ratio_)
    optimal_dim = np.argmax(cumsum_variance >= target_variance) + 1

    # 降维变换矩阵
    transform_matrix = pca.components_[:optimal_dim].T

    return {
        'original_dim': embeddings.shape[1],
        'optimal_dim': optimal_dim,
        'variance_retained': cumsum_variance[optimal_dim - 1],
        'transform_matrix': transform_matrix
    }

# 随机投影降维（适合在线场景）
def random_projection_dim_reduction(
    embeddings,
    target_dim
):
    """
    Johnson-Lindenstrauss引理保证的降维

    JL引理：对于n个点，可以降维到 O(ε^-2 * log n) 维度
    同时保持 (1±ε) 倍的距离
    """
    rp = SparseRandomProjection(n_components=target_dim)
    return rp.fit_transform(embeddings)
```

```sql
-- 向量降维函数（简单截断或平均池化）
CREATE OR REPLACE FUNCTION reduce_vector_dimension(
    p_vector VECTOR,
    p_target_dim INT,
    p_method TEXT DEFAULT 'truncate'  -- 'truncate', 'average'
)
RETURNS VECTOR AS $$
DECLARE
    v_current_dim INT;
    v_result FLOAT[];
    v_chunk_size INT;
    i INT;
    j INT;
    v_sum FLOAT;
BEGIN
    v_current_dim := array_length(p_vector::FLOAT[], 1);

    IF v_current_dim <= p_target_dim THEN
        RETURN p_vector;
    END IF;

    IF p_method = 'truncate' THEN
        -- 简单截断（保留前p_target_dim个元素）
        SELECT ARRAY_AGG(val)
        INTO v_result
        FROM (
            SELECT (p_vector::FLOAT[])[i] AS val
            FROM generate_series(1, p_target_dim) AS i
        ) sub;

    ELSIF p_method = 'average' THEN
        -- 平均池化
        v_chunk_size := v_current_dim / p_target_dim;
        v_result := ARRAY[]::FLOAT[];

        FOR i IN 0..p_target_dim-1 LOOP
            v_sum := 0;
            FOR j IN 1..v_chunk_size LOOP
                v_sum := v_sum + (p_vector::FLOAT[])[i * v_chunk_size + j];
            END LOOP;
            v_result := array_append(v_result, v_sum / v_chunk_size);
        END LOOP;
    END IF;

    RETURN v_result::VECTOR;
END;
$$ LANGUAGE plpgsql;

-- 创建降维后的表

    metadata
FROM documents;
```

### 4.3 批量查询优化

#### 4.3.1 批处理架构

```python
import asyncio
import asyncpg
from typing import List, Tuple
import numpy as np

class BatchVectorSearch:
    def __init__(self, dsn: str, pool_size: int = 20):
        self.dsn = dsn
        self.pool = None
        self.pool_size = pool_size

    async def initialize(self):
        self.pool = await asyncpg.create_pool(
            self.dsn,
            min_size=5,
            max_size=self.pool_size
        )

    async def batch_search(
        self,
        queries: List[List[float]],
        top_k: int = 10,
        batch_size: int = 100
    ) -> List[List[Tuple]]:
        """
        批量向量搜索

        并行处理多个查询，提高吞吐量
        """
        # 分批次处理
        all_results = []
        for i in range(0, len(queries), batch_size):
            batch = queries[i:i + batch_size]

            # 并行执行批次内查询
            tasks = [
                self._single_search(query, top_k)
                for query in batch
            ]
            batch_results = await asyncio.gather(*tasks)
            all_results.extend(batch_results)

        return all_results

    async def _single_search(
        self,
        query_embedding: List[float],
        top_k: int
    ) -> List[Tuple]:
        async with self.pool.acquire() as conn:
            rows = await conn.fetch(
                """
                SELECT id, content, embedding <=> $1::vector AS distance
                FROM documents
                ORDER BY embedding <=> $1::vector
                LIMIT $2
                """,
                query_embedding,
                top_k
            )
            return [(r['id'], r['content'], r['distance']) for r in rows]

    async def optimized_batch_search(
        self,
        queries: List[List[float]],
        top_k: int = 10
    ) -> List[List[Tuple]]:
        """
        优化的批量搜索 - 使用单个查询处理多个向量
        """
        async with self.pool.acquire() as conn:
            # 使用unnest批量处理
            rows = await conn.fetch(
                """
                WITH query_batch AS (
                    SELECT
                        row_number() OVER () AS query_idx,
                        q.embedding AS query_embedding
                    FROM unnest($1::vector[]) AS q(embedding)
                )
                SELECT
                    qb.query_idx,
                    d.id,
                    d.content,
                    d.embedding <=> qb.query_embedding AS distance,
                    row_number() OVER (
                        PARTITION BY qb.query_idx
                        ORDER BY d.embedding <=> qb.query_embedding
                    ) AS rank
                FROM query_batch qb
                CROSS JOIN LATERAL (
                    SELECT id, content, embedding
                    FROM documents
                    ORDER BY embedding <=> qb.query_embedding
                    LIMIT $2
                ) d
                ORDER BY qb.query_idx, rank
                """,
                queries,
                top_k
            )

            # 重组结果
            results = [[] for _ in range(len(queries))]
            for row in rows:
                idx = int(row['query_idx']) - 1
                results[idx].append((
                    row['id'],
                    row['content'],
                    row['distance']
                ))

            return results

# 使用示例
async def main():
    searcher = BatchVectorSearch("postgresql://user:pass@localhost/db")
    await searcher.initialize()

    # 批量查询
    queries = [generate_embedding(f"query {i}") for i in range(1000)]
    results = await searcher.optimized_batch_search(queries, top_k=10)

    await searcher.pool.close()

asyncio.run(main())
```

#### 4.3.2 查询缓存策略

```sql
-- 向量查询结果缓存表
CREATE TABLE vector_query_cache (
    query_hash VARCHAR(64) PRIMARY KEY,  -- SHA256(query_vector)
    query_embedding VECTOR(1536),
    results JSONB,
    hit_count INT DEFAULT 1,
    last_accessed TIMESTAMP DEFAULT NOW(),
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_cache_accessed ON vector_query_cache(last_accessed);

-- 带缓存的向量搜索
CREATE OR REPLACE FUNCTION cached_vector_search(
    p_query_embedding VECTOR(1536),
    p_top_k INT DEFAULT 10,
    p_cache_ttl INTERVAL DEFAULT INTERVAL '5 minutes'
)
RETURNS TABLE (
    doc_id INT,
    content TEXT,
    distance FLOAT
) AS $$
DECLARE
    v_query_hash VARCHAR(64);
    v_cached_results JSONB;
BEGIN
    -- 计算查询哈希
    SELECT encode(digest(p_query_embedding::TEXT, 'sha256'), 'hex')
    INTO v_query_hash;

    -- 检查缓存
    SELECT results INTO v_cached_results
    FROM vector_query_cache
    WHERE query_hash = v_query_hash
      AND created_at > NOW() - p_cache_ttl;

    IF v_cached_results IS NOT NULL THEN
        -- 更新访问统计
        UPDATE vector_query_cache
        SET hit_count = hit_count + 1,
            last_accessed = NOW()
        WHERE query_hash = v_query_hash;

        -- 返回缓存结果
        RETURN QUERY
        SELECT
            (r->>'doc_id')::INT,
            r->>'content',
            (r->>'distance')::FLOAT
        FROM jsonb_array_elements(v_cached_results) AS r;
    ELSE
        -- 执行新查询
        RETURN QUERY
        WITH new_results AS (
            SELECT
                d.id AS doc_id,
                d.content,
                d.embedding <=> p_query_embedding AS distance
            FROM documents d
            ORDER BY d.embedding <=> p_query_embedding
            LIMIT p_top_k
        )
        SELECT * FROM new_results;

        -- 缓存结果
        INSERT INTO vector_query_cache (query_hash, query_embedding, results)
        SELECT
            v_query_hash,
            p_query_embedding,
            jsonb_agg(
                jsonb_build_object(
                    'doc_id', nr.doc_id,
                    'content', nr.content,
                    'distance', nr.distance
                )
            )
        FROM (
            SELECT
                d.id AS doc_id,
                d.content,
                d.embedding <=> p_query_embedding AS distance
            FROM documents d
            ORDER BY d.embedding <=> p_query_embedding
            LIMIT p_top_k
        ) nr
        ON CONFLICT (query_hash) DO UPDATE SET
            results = EXCLUDED.results,
            created_at = NOW();
    END IF;
END;
$$ LANGUAGE plpgsql;
```

```python
class DocumentKnowledgeBase:

        ON kb_documents USING GIN(tags);
        """

        cur = self.conn.cursor()
        cur.execute(sql)
        self.conn.commit()
        cur.close()

    def chunk_document(self, content: str, chunk_size: int = 1000, overlap: int = 200):
        """
        文档分块

        使用滑动窗口进行语义分块
        """
        chunks = []
        start = 0

        while start < len(content):
            end = min(start + chunk_size, len(content))

            # 尽量在句子边界分割
            if end < len(content):
                # 向后查找最近的句子结束符
                for i in range(end, max(start + chunk_size - 100, start), -1):
                    if content[i-1] in '.。!！?？\n':
                        end = i
                        break

            chunk = content[start:end].strip()
            if chunk:
                chunks.append({
                    'content': chunk,
                    'start': start,
                    'end': end
                })

            start = end - overlap

        return chunks

    def add_document(self, doc_id: str, title: str, content: str,
                     metadata: dict = None, generate_embeddings: bool = True):
        """添加文档到知识库"""
        cur = self.conn.cursor()

        # 分块处理
        chunks = self.chunk_document(content)

        # 插入主文档（摘要）
        if generate_embeddings:
            summary_embedding = generate_embedding(title + " " + content[:2000])
        else:
            summary_embedding = None

        cur.execute("""
            INSERT INTO kb_documents
            (doc_id, title, content, content_type, embedding, metadata)
            VALUES (%s, %s, %s, %s, %s, %s)
            RETURNING id
        """, (doc_id, title, content, 'text', summary_embedding,
              json.dumps(metadata or {})))

        parent_id = cur.fetchone()[0]

        # 插入分块
        for i, chunk in enumerate(chunks):
            if generate_embeddings:
                chunk_embedding = generate_embedding(chunk['content'])
            else:
                chunk_embedding = None

            cur.execute("""
                INSERT INTO kb_documents
                (doc_id, title, content, content_type, embedding,
                 chunk_parent_id, chunk_index, metadata)
                VALUES (%s, %s, %s, %s, %s, %s, %s, %s)
            """, (f"{doc_id}_chunk_{i}", title, chunk['content'], 'chunk',
                  chunk_embedding, parent_id, i,
                  json.dumps({'start': chunk['start'], 'end': chunk['end']})))

        self.conn.commit()
        cur.close()

    def semantic_search(self, query: str, filters: dict = None, top_k: int = 10):
        """语义搜索"""
        query_embedding = generate_embedding(query)

        cur = self.conn.cursor()

        # 构建过滤条件
        where_clauses = []
        params = [query_embedding, top_k]

        if filters:
            if 'category' in filters:
                where_clauses.append("category = %s")
                params.insert(-1, filters['category'])
            if 'tags' in filters:
                where_clauses.append("tags && %s")
                params.insert(-1, filters['tags'])

        where_sql = " AND ".join(where_clauses) if where_clauses else "TRUE"

        cur.execute(f"""
            SELECT
                d.id,
                d.doc_id,
                d.title,
                SUBSTRING(d.content, 1, 500) AS content_preview,
                1 - (d.embedding <=> %s::vector) AS similarity,
                d.category,
                d.tags
            FROM kb_documents d
            WHERE {where_sql}
              AND chunk_parent_id IS NULL
            ORDER BY d.embedding <=> %s::vector
            LIMIT %s
        """, params)

        return cur.fetchall()
```

### 5.3 智能客服系统

```sql
-- 智能客服问答系统

-- 问答对表
CREATE TABLE faq_entries (
    id SERIAL PRIMARY KEY,
    question TEXT NOT NULL,
    answer TEXT NOT NULL,
    question_embedding VECTOR(768),
    answer_embedding VECTOR(768),
    category VARCHAR(100),
    tags TEXT[],
    hit_count INT DEFAULT 0,
    helpful_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 对话会话表
CREATE TABLE chat_sessions (
    session_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id VARCHAR(100),
    context_embedding VECTOR(768),  -- 会话上下文嵌入
    history JSONB DEFAULT '[]',
    created_at TIMESTAMP DEFAULT NOW(),
    last_activity TIMESTAMP DEFAULT NOW()
);

-- 智能FAQ检索
CREATE OR REPLACE FUNCTION intelligent_faq_search(
    p_query TEXT,
    p_query_embedding VECTOR(768),
    p_session_id UUID DEFAULT NULL,
    p_category VARCHAR(100) DEFAULT NULL,
    p_threshold FLOAT DEFAULT 0.7,
    p_top_k INT DEFAULT 5
)
RETURNS TABLE (
    faq_id INT,
    question TEXT,
    answer TEXT,
    similarity FLOAT,
    confidence_level VARCHAR(20)
) AS $$
DECLARE
    v_session_context VECTOR(768);
BEGIN
    -- 获取会话上下文
    IF p_session_id IS NOT NULL THEN
        SELECT context_embedding INTO v_session_context
        FROM chat_sessions
        WHERE session_id = p_session_id;
    END IF;

    RETURN QUERY
    SELECT
        f.id,
        f.question,
        f.answer,
        CASE
            WHEN v_session_context IS NOT NULL THEN
                0.7 * (1 - (f.question_embedding <=> p_query_embedding)) +
                0.3 * (1 - (f.question_embedding <=> v_session_context))
            ELSE
                1 - (f.question_embedding <=> p_query_embedding)
        END AS similarity,
        CASE
            WHEN 1 - (f.question_embedding <=> p_query_embedding) >= 0.9 THEN 'HIGH'
            WHEN 1 - (f.question_embedding <=> p_query_embedding) >= 0.75 THEN 'MEDIUM'
            ELSE 'LOW'
        END AS confidence_level
    FROM faq_entries f
    WHERE (p_category IS NULL OR f.category = p_category)
      AND 1 - (f.question_embedding <=> p_query_embedding) >= p_threshold
    ORDER BY f.question_embedding <=> p_query_embedding
    LIMIT p_top_k;
END;
$$ LANGUAGE plpgsql;

-- 更新FAQ命中率
CREATE OR REPLACE FUNCTION record_faq_hit(p_faq_id INT)
RETURNS VOID AS $$
BEGIN
    UPDATE faq_entries
    SET hit_count = hit_count + 1,
        updated_at = NOW()
    WHERE id = p_faq_id;
END;
$$ LANGUAGE plpgsql;
```
