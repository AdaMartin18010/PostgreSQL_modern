## 证据

### 1. 官方发布信息

> PostgreSQL 18 adds **temporal constraints** -- constraints over ranges -- for both `PRIMARY KEY` and `UNIQUE` constraints using the `WITHOUT OVERLAPS` clause, and on `FOREIGN KEY` constraints using the `PERIOD` clause.

### 2. 官方 Release Notes 摘录

来源: <https://www.postgresql.org/docs/release/18.0/>

```
Allow the specification of non-overlapping PRIMARY KEY, UNIQUE, and foreign key constraints (Paul A. Jungwirth)

This is specified by WITHOUT OVERLAPS for PRIMARY KEY and UNIQUE, and by PERIOD for foreign keys, all applied to the last specified column.
```

### 3. 官方文档链接

**CREATE TABLE 文档** (当前版本):

- <https://www.postgresql.org/docs/current/sql-createtable.html>

**UNIQUE 约束语法**:

```sql
UNIQUE [ NULLS [ NOT ] DISTINCT ] ( column_name [, ... ] [, column_name WITHOUT OVERLAPS ] )
```

> If the `WITHOUT OVERLAPS` option is specified for the last column, then that column is checked for overlaps instead of equality. In that case, the other columns of the constraint will allow duplicates so long as the duplicates don't overlap in the `WITHOUT OVERLAPS` column.

**FOREIGN KEY 约束语法**:

```sql
FOREIGN KEY ( column_name [, ... ] [, PERIOD column_name ] )
REFERENCES reftable [ ( refcolumn [, ... ] [, PERIOD refcolumn ] ) ]
```

> If the last column is marked with `PERIOD`, it is treated in a special way. The constraint is considered satisfied if the referenced table has matching records whose combined `PERIOD` values completely cover the referencing record's.

## 功能详情

### 支持的约束类型

| 约束类型 | 语法 | 说明 |
|---------|------|------|
| PRIMARY KEY | `PRIMARY KEY (col1, range_col WITHOUT OVERLAPS)` | 时态主键 |
| UNIQUE | `UNIQUE (col1, range_col WITHOUT OVERLAPS)` | 时态唯一约束 |
| FOREIGN KEY | `FOREIGN KEY (col1, PERIOD range_col) REFERENCES ...` | 时态外键 |

### 支持的数据类型

- 范围类型: `TSTZRANGE`, `TSRANGE`, `DATERANGE`, `INT4RANGE`, `INT8RANGE`, `NUMRANGE`
- 多范围类型 (Multirange)

### 实现机制

- 内部使用 `EXCLUDE USING GIST` 约束实现
- 需要 `btree_gist` 扩展用于非范围类型的键列
- 使用 GiST 索引进行高效的重叠检测
