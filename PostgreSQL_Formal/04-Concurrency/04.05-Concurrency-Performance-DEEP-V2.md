## 1. 性能理论基础

### 1.1 Little定律

**定理 1.1 (Little定律)**:

$$
L = \lambda \cdot W
$$

其中:

- $L$: 系统中平均请求数
- $\lambda$: 平均到达率 (请求/秒)
- $W$: 平均等待时间 (秒)

*证明*: 考虑系统在时间$T$内处理的请求。到达数 = $\lambda T$，总等待时间 = $L T$。因此平均等待时间 $W = L/\lambda$。∎

### 1.2 吞吐量与延迟

**定义 1.1 (吞吐量)**:

$$
\text{Throughput} = \frac{\text{Completed Transactions}}{\text{Time}}
$$

**定义 1.2 (延迟)**:

$$
\text{Latency} = T_{queue} + T_{service}
$$

**定理 1.2 (延迟-吞吐量权衡)**: 在饱和点之前，增加负载提高吞吐量但增加延迟。

## 2. 锁竞争模型

### 2.1 冲突概率

**定义 2.1 (冲突概率)**:
设数据库有$D$个数据项，事务访问$k$个数据项，$n$个并发事务。

$$
P_{conflict} = 1 - \left(1 - \frac{k}{D}\right)^{n-1} \approx \frac{k(n-1)}{D}
$$

**定理 2.1 (死锁概率)**: 死锁概率随并发度指数增长。

$$
P_{deadlock} \propto n^2 \cdot k^2 \cdot \frac{1}{D}
$$

### 2.2 等待时间分析

**M/M/1队列模型**:

$$
W = \frac{1}{\mu - \lambda}
$$

其中$\mu$是服务率，$\lambda$是到达率。

**定理 2.2 (饱和点)**: 当$\lambda \to \mu$时，$W \to \infty$。

## 3. MVCC性能分析

### 3.1 版本链开销

**定义 3.1 (版本链长度)**:

$$
L_{chain} = \lambda \cdot T_{retention}
$$

**定理 3.1 (读取性能衰减)**: 随着版本链增长，读取性能线性下降。

$$
T_{read} = T_{base} + c \cdot L_{chain}
$$

### 3.2 清理开销

**定义 3.2 (死元组比例)**:

$$
r_{dead} = \frac{N_{dead}}{N_{live} + N_{dead}}
$$

**定理 3.2 (VACUUM必要性)**: 当$r_{dead} > 0.2$时，需要执行VACUUM。

## 4. 连接池优化

### 4.1 最优连接数

**定义 4.1 (最优连接数)**:

$$
N_{optimal} = N_{cores} \times (1 + \frac{W}{S})
$$

其中$W$是等待时间，$S$是服务时间。

**定理 4.1 (连接过多惩罚)**: 超过最优连接数会导致上下文切换开销增加。

## 5. 工作负载分类

### 5.1 OLTP优化

**特征**:

- 短事务
- 随机访问
- 高并发

**优化策略**:

```sql
-- 使用连接池
-- 准备语句
PREPARE insert_order (INT, INT, NUMERIC) AS
    INSERT INTO orders (user_id, product_id, amount)
    VALUES ($1, $2, $3);

-- 批量提交
BEGIN;
EXECUTE insert_order(1, 100, 99.99);
EXECUTE insert_order(2, 101, 49.99);
COMMIT;
```

### 5.2 OLAP优化

**特征**:

- 长查询
- 顺序扫描
- 低并发

**优化策略**:

```sql
-- 使用并行查询
SET max_parallel_workers_per_gather = 4;

-- 调整work_mem
SET work_mem = '1GB';
```

## 6. 监控与诊断

### 6.1 关键指标

```sql
-- 事务统计
SELECT
    xact_commit,
    xact_rollback,
    blks_hit,
    blks_read,
    ROUND(100.0 * blks_hit / (blks_hit + blks_read), 2) AS cache_hit_ratio
FROM pg_stat_database;

-- 锁等待
SELECT
    wait_event_type,
    wait_event,
    count(*)
FROM pg_stat_activity
WHERE wait_event_type IS NOT NULL
GROUP BY 1, 2;

-- 冲突统计
SELECT
    confl_tablespace,
    confl_lock,
    confl_snapshot,
    confl_deadlock
FROM pg_stat_database_conflicts;
```

### 6.2 性能调优矩阵

| 问题 | 症状 | 解决方案 |
|------|------|----------|
| 高冲突率 | 大量锁等待 | 减少事务大小，优化访问顺序 |
| 低缓存命中率 | 高I/O | 增加shared_buffers |
| 高回滚率 | 频繁死锁 | 统一加锁顺序 |
| 长查询 | 连接堆积 | 连接池，查询优化 |

## 8. 实例与反例

### 8.1 正例

**场景1: 连接池优化**:

```python
# 使用连接池
from psycopg2 import pool

try:
    connection_pool = pool.ThreadedConnectionPool(
        5,  # 最小连接数
        20, # 最大连接数
        host='localhost',
        database='mydb',
        user='user',
        password='pass'
    )
except:
    print("Error creating connection pool")

# 使用连接
conn = connection_pool.getconn()
# ... 执行查询 ...
connection_pool.putconn(conn)
```

**场景2: 批量插入**:

```sql
-- 批量插入减少事务开销
BEGIN;
INSERT INTO logs VALUES (1, 'msg1'), (2, 'msg2'), ...;
COMMIT;

-- vs 单条插入 (慢10倍)
BEGIN; INSERT VALUES (1); COMMIT;
BEGIN; INSERT VALUES (2); COMMIT;
```
